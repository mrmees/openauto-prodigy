  <name>Task 1: Snap-aware grid computation + tighter card padding</name>
  <files>qml/applications/home/HomeMenu.qml</files>
  <action>
Replace the grid frame computation block (lines 21-28) with a two-stage snap-aware version:

1. **Add `baseCellSide` and `kSnapThreshold` properties:**
   - `readonly property real baseCellSide: DisplayInfo ? DisplayInfo.cellSide : 120`
   - `readonly property real kSnapThreshold: 0.6` (non-configurable QML constant)

2. **Compute initial grid from base cell size:**
   - `_baseCols = Math.max(3, Math.floor(pageView.width / baseCellSide))` (guard pageView.width > 0)
   - `_baseRows = Math.max(2, Math.floor(pageView.height / baseCellSide))` (guard pageView.height > 0)

3. **Apply snap threshold:**
   - `_xWaste = pageView.width - _baseCols * baseCellSide`
   - `_yWaste = pageView.height - _baseRows * baseCellSide`
   - `gridCols = _xWaste > kSnapThreshold * baseCellSide ? _baseCols + 1 : _baseCols`
   - `gridRows = _yWaste > kSnapThreshold * baseCellSide ? _baseRows + 1 : _baseRows`

4. **Derive effective cellSide (square cells):**
   - `cellSide = Math.min(pageView.width / gridCols, pageView.height / gridRows)`
   - gridW, gridH, offsetX, offsetY computed from effective cellSide (same formulas as before)

5. **Keep `cellWidth` and `cellHeight` legacy aliases** pointing to the new `cellSide`.

6. **CRITICAL: Avoid binding loop.** `baseCellSide` (from DisplayInfo) feeds into `_baseCols`/`_baseRows`, then snap produces `gridCols`/`gridRows`, then `cellSide` is derived from those. Two-stage, no circularity.

7. **Tighten card padding** at line 280:
   - Change `anchors.margins: UiMetrics.spacing / 2` to `anchors.margins: Math.max(UiMetrics.spacing / 4, 2)`
   - This halves the visual gap between cards (~8px total to ~4px total) while maintaining minimum 2px to avoid glass card radius clipping.

The existing `onGridColsChanged` and `onGridRowsChanged` handlers (lines 38-49) already call `WidgetGridModel.setGridDimensions()` — no changes needed there. The snap will trigger a dimension change (e.g., 7x3 to 7x4) which invokes the remap system automatically.
