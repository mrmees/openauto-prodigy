**Part A: NavbarControl.qml — Add page dots flanking the clock**

The clock currently renders as standalone Text (horizontal, line 85) or Column (vertical, line 95). Replace each with a layout that includes dots on either side of the clock.

1. **Add computed properties** at the top of the NavbarControl Item:
   ```javascript
   // Page dot data — only when showing clock on home screen
   readonly property bool showDots: root.showClock
       && WidgetGridModel.pageCount > 1
       && !PluginModel.activePluginId
       && ApplicationController.currentApplication !== 6
   readonly property int leftDotCount: showDots ? WidgetGridModel.activePage : 0
   readonly property int rightDotCount: showDots ? Math.max(0, WidgetGridModel.pageCount - WidgetGridModel.activePage - 1) : 0
   readonly property real dotSize: UiMetrics.spacing * 1.2
   ```

2. **Horizontal clock (replace lines 84-92):** Wrap `clockHoriz` in a `Row` with dot Repeaters on each side:
   ```qml
   Row {
       anchors.centerIn: parent
       spacing: UiMetrics.spacing
       visible: root.showClock && !root.isVertical

       Repeater {
           model: root.leftDotCount
           Rectangle {
               width: root.dotSize; height: root.dotSize; radius: root.dotSize / 2
               color: ThemeService.onSurfaceVariant; opacity: 0.4
               anchors.verticalCenter: parent ? parent.verticalCenter : undefined
           }
       }

       Text {
           id: clockHoriz
           color: navbar.barFg
           font.pixelSize: Math.round(root.height * 0.75)
           font.weight: Font.DemiBold
           anchors.verticalCenter: parent ? parent.verticalCenter : undefined
       }

       Repeater {
           model: root.rightDotCount
           Rectangle {
               width: root.dotSize; height: root.dotSize; radius: root.dotSize / 2
               color: ThemeService.onSurfaceVariant; opacity: 0.4
               anchors.verticalCenter: parent ? parent.verticalCenter : undefined
           }
       }
   }
   ```

3. **Vertical clock (replace lines 94-112):** Wrap `clockVertRepeater` Column in a parent `Column` with dot Repeaters above and below:
   ```qml
   Column {
       anchors.centerIn: parent
       spacing: UiMetrics.spacing
       visible: root.showClock && root.isVertical

       Repeater {
           model: root.leftDotCount  // "left" = above in vertical
           Rectangle {
               width: root.dotSize; height: root.dotSize; radius: root.dotSize / 2
               color: ThemeService.onSurfaceVariant; opacity: 0.4
               anchors.horizontalCenter: parent ? parent.horizontalCenter : undefined
           }
       }

       Column {
           anchors.horizontalCenter: parent ? parent.horizontalCenter : undefined
           Repeater {
               id: clockVertRepeater
               model: []
               Text {
                   text: modelData
                   font.pixelSize: Math.round(root.width * 0.55)
                   font.weight: Font.DemiBold
                   color: navbar.barFg
                   horizontalAlignment: Text.AlignHCenter
                   width: root.width
               }
           }
       }

       Repeater {
           model: root.rightDotCount  // "right" = below in vertical
           Rectangle {
               width: root.dotSize; height: root.dotSize; radius: root.dotSize / 2
               color: ThemeService.onSurfaceVariant; opacity: 0.4
               anchors.horizontalCenter: parent ? parent.horizontalCenter : undefined
           }
       }
   }
   ```

4. **Clock timer** (lines 56-82) stays unchanged — it still updates `clockHoriz.text` and `clockVertRepeater.model`. The `clockHoriz` ID must remain the same since the timer references it.

5. **DO NOT make dots tappable** — no MouseArea on dots. The existing MouseArea at line 115 handles the entire control's touch gestures; dots are purely visual.

**Part B: HomeMenu.qml — Delete PageIndicator block**

Delete lines 532-557 (the entire `PageIndicator` block including the `onCurrentIndexChanged` handler). The ColumnLayout spacing between SwipeView and the now-deleted PageIndicator will have no effect (spacing only applies between visible children). The reverse sync (pageView -> WidgetGridModel.activePage) is handled by HomeMenu.qml line ~119 and is NOT in the PageIndicator block.
