Create the `docs/pi-config/kiosk/` directory structure with 5 config files and a README. These are the reference configs that Phase 32's installer integration will deploy to system paths.

**1. `docs/pi-config/kiosk/openauto-kiosk.desktop`** -- XDG session entry file:
```ini
[Desktop Entry]
Name=OpenAuto Kiosk
Comment=OpenAuto Prodigy kiosk session
Exec=labwc -C /etc/openauto-kiosk/labwc
Type=Application
DesktopNames=labwc
```
- Deploys to `/usr/share/wayland-sessions/openauto-kiosk.desktop`
- The `-C` flag is the key architectural decision: labwc reads ALL config from `/etc/openauto-kiosk/labwc/` instead of `~/.config/labwc/`
- `DesktopNames=labwc` -- matches the compositor identity for XDG portal routing
- Do NOT include `NoDisplay=true` (LightDM needs to see it for session selection)
- No `-s` flag for autostart -- labwc reads autostart from the `-C` directory automatically

**2. `docs/pi-config/kiosk/labwc/rc.xml`** -- Stripped kiosk labwc config:
```xml
<?xml version="1.0"?>
<labwc_config>
  <touch deviceName="" mapToOutput="" mouseEmulation="no"/>
  <core>
    <decoration>server</decoration>
    <gap>0</gap>
  </core>
  <theme>
    <cornerRadius>0</cornerRadius>
    <keepBorder>no</keepBorder>
  </theme>
  <windowRules>
    <windowRule identifier="openauto*" serverDecoration="no">
      <action name="ToggleFullscreen"/>
    </windowRule>
  </windowRules>
</labwc_config>
```
- Deploys to `/etc/openauto-kiosk/labwc/rc.xml`
- `mouseEmulation="no"` is CRITICAL for multi-touch (existing requirement, Pitfall 8 prevention)
- `identifier="openauto*"` uses wildcard per Claude's discretion decision in CONTEXT.md -- avoids app_id mismatch (Pitfall 15) without requiring main.cpp changes
- `serverDecoration="no"` removes title bar
- `ToggleFullscreen` rule makes the app start fullscreen automatically
- `gap=0`, `cornerRadius=0`, `keepBorder=no` -- no visual chrome
- Root element is `<labwc_config>` (NOT `<openbox_config>` -- labwc accepts both but `labwc_config` is canonical). Actually, checking the existing config at docs/pi-config/labwc-rc.xml which uses `<openbox_config>` -- use whichever the research docs show. The ARCHITECTURE.md examples use `<labwc_config>` -- use that for the new kiosk config to match the research reference.

IMPORTANT: Re-read ARCHITECTURE.md and STACK.md for the exact rc.xml format. The STACK.md example uses `<openbox_config xmlns="http://openbox.org/3.4/rc">` while ARCHITECTURE.md uses `<labwc_config>`. The existing desktop config (`docs/pi-config/labwc-rc.xml`) uses `<openbox_config>`. For maximum compatibility, use `<openbox_config xmlns="http://openbox.org/3.4/rc">` since that matches what already works on the Pi.

Remove this line (currently ~line 1631):
```
ExecStartPre=-/bin/sh -c 'systemctl --user stop wf-panel-restore.service 2>/dev/null; pkill -f "lwrespawn.*wf-panel-pi"; pkill wf-panel-pi; true'
```

Remove this line (currently ~line 1634):
```
ExecStopPost=-/bin/sh -c '[ "\$SERVICE_RESULT" = "success" ] && systemd-run --user --unit=wf-panel-restore --setenv=WAYLAND_DISPLAY=wayland-0 /usr/bin/lwrespawn /usr/bin/wf-panel-pi || true'
```

The resulting service template should have these lines in order (within the heredoc):
- `ExecStartPre=+/usr/local/bin/openauto-preflight` (keep -- preflight is still needed)
- `ExecStart=$INSTALL_DIR/build/src/openauto-prodigy` (keep)
- `ExecStopPost=-/bin/sh -c '[ "\$SERVICE_RESULT" = "success" ] && timeout 5 /usr/bin/bluetoothctl disconnect || true'` (keep -- BT disconnect on clean stop)
- The wf-panel-pi ExecStartPre and ExecStopPost lines are GONE

Per CONTEXT.md decision: "One service file for both modes -- no panel management lines in either. Drop the wf-panel-pi kill/restore entirely. Kiosk mode provides the clean experience; desktop mode users accept the panel stays visible."

Do NOT delete the function -- it's called from the step orchestration flow. Just update its message.
