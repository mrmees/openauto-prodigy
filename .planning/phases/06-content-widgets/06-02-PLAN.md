<objective>
Build MediaDataBridge: a unified QObject bridge that merges AA media metadata and BT A2DP metadata with deterministic source priority (AA > BT > None). Exposes unified Q_PROPERTYs for title, artist, playback state, and source. Delegates playback control Q_INVOKABLEs to the correct backend (AVRCP for BT, sendButtonPress for AA).

Purpose: The unified Now Playing widget (Plan 03) needs a single set of QML bindings regardless of media source. This bridge is the single source of truth.
Output: One tested C++ class ready for QML consumption.

    - AA metadata signal updates title/artist when source is AA
    - When AA disconnects: source reverts to Bluetooth if BT connected, else None. BT metadata read immediately from plugin properties (no stale AA data flash).
    - BT metadata signals while source is AA are stored internally but not surfaced (AA takes priority)
    - When source is BT and BT disconnects: source becomes None, metadata cleared
    - playPause: when source=AA calls orchestrator->sendButtonPress(85), when source=BT calls btPlugin->play() or pause() based on playbackState
    - next: when source=AA calls sendButtonPress(87), when source=BT calls btPlugin->next()
    - previous: when source=AA calls sendButtonPress(88), when source=BT calls btPlugin->previous()
    - When source=None, controls do nothing (no crash)
    - albumArt bytes stored in private member (not exposed as Q_PROPERTY) for future POLISH-02
    - playbackState reflects current source's state (0=stopped, 1=playing, 2=paused)
    - appName stored from AA playback state changes (exposed as Q_PROPERTY for source indicator)

      - Connect to bt->metadataChanged — store BT metadata, update properties if source==BT
      - Connect to bt->playbackStateChanged — store BT playback state
      - Connect to bt->connectionStateChanged — if BT disconnects while source==BT, switch to None
      - Store bt pointer for AVRCP control delegation
      - Read current BT state immediately (bt->trackTitle() etc) to handle case where BT was already connected
    - `playPause()`, `next()`, `previous()` Q_INVOKABLEs — switch on source_, delegate to correct backend
    - On source switch: immediately read current metadata from new source (BT properties or cached AA values). Emit metadataChanged, sourceChanged, playbackStateChanged.
    - Private storage: separate aaTitle_/btTitle_/aaArtist_/btArtist_ etc. so switching sources reads the right cache.
    - albumArt_: QByteArray stored from AA metadataChanged, never exposed to QML.
