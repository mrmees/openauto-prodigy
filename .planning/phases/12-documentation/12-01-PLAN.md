**Part A: Extend `docs/design-decisions.md`** with a new section "## v0.6-v0.6.1: Widget Framework and Grid Architecture" containing ADRs for these decisions from STATE.md:

1. **DPI-based grid sizing** â€” `cellSide = diagPx / (9.0 + bias * 0.8)`. Decision + rationale (resolution-independent, DPI cascade is scaffolding for future mm-based path).
2. **Grid cols/rows computed in QML** â€” reactive QML bindings from DisplayInfo.cellSide, not C++ computation. Rationale: stays reactive if window resizes.
3. **Dock replaced by singleton widgets** â€” z=10 overlay outside ColumnLayout replaced by singleton launcher widgets on reserved page. Rationale: reclaim vertical space, unified widget model.
4. **JS-based category filtering in QML** â€” not C++ proxy model. Rationale: sufficient for small widget counts, simpler.
5. **Category order hardcoded** â€” static map (status=0, media=1, navigation=2, launcher=3). Rationale: simple, matches spec.
6. **Remap clamps oversized spans** â€” instead of hiding widgets. Rationale: maximizes widget visibility after grid dimension changes.
7. **promoteToBase() on every mutation** â€” live always reflects latest user intent as base. Rationale: remap derives from base, so base must stay current.
8. **Base/live snapshot pattern** â€” basePlacements_ from YAML, livePlacements_ is runtime; remap derives from base. Rationale: clean separation of persisted vs runtime state.
9. **Reserved page from singleton presence** â€” pageHasSingleton check, not explicit page flag. Rationale: derived state is more robust than stored state.
10. **Fixed instanceIds for seeded singletons** â€” deterministic IDs like "aa-launcher-reserved". Rationale: prevents duplicate seeding on restart.
11. **Auto-snap threshold** â€” 50% waste (`kSnapThreshold: 0.5`) against baseCellSide, two-pass with cascade guard. Pass 1 snaps axes with >50% waste. Pass 2 catches cascaded waste from pass 1's cellSide shrink, but only on axes that did NOT snap in pass 1 (prevents iterative packing like 7x4â†’8x5â†’9x5). Rationale: recovers gutter space without runaway iteration. Source: HomeMenu.qml lines 26-50.
12. **Clock as active page indicator** â€” leftDotCount = activePage, rightDotCount = pageCount - activePage - 1. Rationale: repurposes existing navbar element.
13. **WidgetContextFactory as dedicated class** â€” not on WidgetGridModel. Rationale: keeps model pure data, factory owns IHostContext + cell geometry.
14. **Context injection via Loader.onLoaded + Binding** â€” not context properties. Rationale: typed, documented, NOTIFY-capable.
15. **NowPlayingWidget controls as provider methods** â€” not ActionRegistry candidates. Rationale: transport controls are media-specific, not general actions.

Follow the existing format in design-decisions.md: section header with decision name, then narrative paragraphs explaining the decision and its rationale. Group related decisions under subsections.
