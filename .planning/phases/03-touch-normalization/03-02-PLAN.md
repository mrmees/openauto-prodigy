**DebugSettings.qml restructure:**

The Debug page has the most complex layout. Restructure as follows:

**Section "Video Decoding":**
- Each codec from the Repeater gets a SettingsRow. The codec enable/disable row (codec name + Switch) is a SettingsRow with the Repeater's index as rowIndex.
- Sub-rows (Decoder hw/sw toggle, Decoder name dropdown) are also SettingsRows but with additional left margin indentation (`anchors.leftMargin: UiMetrics.marginRow * 3` inside the SettingsRow content). Sub-rows are `visible:` only when the parent codec Switch is checked. Sub-row rowIndex continues from parent (parent=0, hw/sw=1, decoder=2, next codec=3, etc). Since visibility changes, accept that hidden rows may cause visual gaps in alternation -- this is fine, the alternation is per-visible-position feel, not strict.
- Remove the separator Rectangles between codecs (SettingsRow alternation provides visual separation).
- For the decoder dropdown row: change `fontCaption` to `fontBody` if any caption text exists. The "Decoder name" label should be `fontBody`. The decoder value text stays `fontBody` with `onSurfaceVariant` color.

**Section "Logging":**
- Reset counter. Wrap Verbose Logging toggle in SettingsRow rowIndex 0.

**Section "Protocol Capture":**
- Reset counter. Wrap Enable Capture toggle (rowIndex 0), Format SegmentedButton (rowIndex 1), Include Media Frames toggle (rowIndex 2), Capture Path ReadOnlyField (rowIndex 3) in SettingsRows.

**Section "Connection Info":**
- Reset counter. Wrap TCP Port ReadOnlyField in SettingsRow rowIndex 0.

**Section "WiFi Access Point":**
- Reset counter. Remove the helper Text ("Configured at install time. Edit YAML config to change.").
- Wrap Channel ReadOnlyField (rowIndex 0) and Band ReadOnlyField (rowIndex 1) in SettingsRows.

**Section "AA Protocol Test":**
- Remove the helper Text ("Test outbound commands (requires active AA connection)").
- Remove the entire button grid (both RowLayouts with ElevatedButtons).
- Remove the status Text at the bottom.
- Replace with a single interactive SettingsRow (rowIndex 0, interactive: true) labeled "AA Protocol Test" with a chevron icon. Clicking it pushes a sub-page view.
- The sub-page approach: Use the page's own StackView or the parent StackView. Since settings pages are loaded inside a StackView in the settings navigation, the simplest approach is to toggle visibility between the main content and a test panel within the same Flickable. Add an `inTestMode` property. When true, show the button grid (both rows of ElevatedButtons) and a back row. When false, show the normal settings content.
- Alternative simpler approach: Just inline the button grid below the row, collapsed/expanded on tap (like an accordion). The SettingsRow tap toggles a `showTestButtons` boolean. When true, the button grid rows appear below. This avoids StackView complexity.
- Use the accordion approach: tap the "AA Protocol Test" SettingsRow to toggle `showTestButtons`. Button grid rows appear/disappear below it. The AA connection status indicator (connected/not) can be a small status dot in the SettingsRow itself (green dot if connected, gray if not).
