    Add a Q_INVOKABLE method to ThemeService:

    ```cpp
    Q_INVOKABLE bool isUserTheme(const QString& themeId) const;
    ```
    Implementation: look up themeId in themeDirectories_. Check if the directory path starts with `QDir::homePath() + "/.openauto/themes/"`. Return true if yes (user/companion theme), false if bundled. This lets QML decide whether to show a delete button.

    **Part B — DisplaySettings.qml: add delete action to theme picker**

    The current FullScreenPicker for "Theme" (line 162) shows a list of theme names. After the picker, add a conditional delete button that appears when the currently selected theme is a user theme.

    Replace the simple FullScreenPicker with a Row or Item containing:
    1. The existing FullScreenPicker (takes most of the width)
    2. A delete button (trash icon or "X") that is visible only when `ThemeService.isUserTheme(ThemeService.availableThemes[currentIndex])` is true

    The delete button calls `ThemeService.deleteTheme(themeId)` when tapped. After deletion, the picker's model updates automatically via availableThemesChanged signal.

    Alternative simpler approach: Add a separate "Delete Theme" item below the picker that is only visible when the current theme is a user theme. Uses a SettingsListItem or custom row with the current theme name and a delete action. This avoids modifying the FullScreenPicker component itself.

    Use ThemeService color properties for styling (no hardcoded colors). Use UiMetrics for sizing (no hardcoded pixel values). The delete action should have a confirmation — either a second tap ("Tap again to confirm") or use ThemeService.red color to indicate destructive action.

    Implementation choice is at Claude's discretion — pick whichever approach integrates cleanly with the existing QML patterns.
