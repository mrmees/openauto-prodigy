  <files>system-service/proxy_manager.py</files>
  <action>
Modify ProxyManager to add honest status verification. Key changes:

1. **Add asyncio.Lock** — single `self._op_lock` guarding enable(), disable(), get_status(), and health loop tick. Lock timeout: 10 seconds. **Critical: enable() calls disable() internally (reconfigure at line 93, rollback at line 128). The lock must NOT be acquired at the top of enable()/disable() directly. Instead, use an internal `_enable_locked()`/`_disable_locked()` pattern: the public methods acquire the lock, then call the lockless internal methods. Internal cross-calls (enable→disable, rollback) call the lockless variant to avoid self-deadlock.**

2. **Add injectable verification methods** (mockable seam for tests):
   - `async def _verify_listener(self) -> bool` — TCP connect to `127.0.0.1:{self._redirect_port}` with 2s timeout. Returns True if connection succeeds. This is the listener truth gate per user decision (not PID).
   - `async def _verify_routing(self) -> bool` — Run `iptables -t nat -L OPENAUTO_PROXY -n` and verify: (a) chain exists (rc==0), (b) output contains REDIRECT rule with `--to-ports {redirect_port}`. Then verify OUTPUT jump exists via `iptables -t nat -L OUTPUT -n` containing `OPENAUTO_PROXY`. Returns True if all present.
   - `async def _verify_upstream(self) -> bool` — TCP connect to `self._proxy_host:self._proxy_port` with 5s timeout. Returns True on success.
   - `async def _verify_all(self) -> dict` — Calls all three, returns `{"listener_ok": bool, "routing_ok": bool, "upstream_ok": bool, "error_code": str|None, "error": str|None}`. Error code priority: `listener_down` > `routing_missing` > `upstream_unreachable`.

3. **Add `get_status()` method** — returns the full response shape:
   ```python
   async def get_status(self, *, verify: bool = False) -> dict:
   ```
   - Acquires `_op_lock` with 10s timeout.
   - If `verify=True`, runs `_verify_all()` and updates cached state.
   - Returns: `{"state": str, "checks": {"listener": bool, "iptables": bool, "upstream": bool}, "error_code": str|None, "error": str|None, "verified_at": str|None, "live_check": bool}`
   - On lock timeout: return cached state with `error_code: "operation_timeout"`.

4. **Modify enable()** — after iptables apply + dns enable, run `_verify_all()`. If listener or routing check fails, set FAILED (not ACTIVE). Only set ACTIVE if local pipeline verified. Store verification result in cached status.

5. **Redesign _health_loop()** — 10s interval (not 30s). Each tick acquires `_op_lock`, runs `_verify_all()`, updates state:
   - Local failure (listener or routing) -> immediate FAILED (no threshold per user decision)
   - Local OK + upstream bad -> DEGRADED
   - All OK -> ACTIVE
   - No auto-recovery actions (do NOT restart redsocks, do NOT reapply iptables — detect and report only per user decision)
   - DO detect external recovery: if state was FAILED and verification now passes (someone manually fixed the pipeline), transition back to ACTIVE. "No auto-recovery" = no corrective action, not ignoring evidence of health.

6. **Add cached status fields**: `_cached_checks`, `_cached_error_code`, `_cached_error`, `_cached_verified_at` (ISO 8601 string). Updated by every verification pass.

7. **Logging policy**: healthy steady-state = no output. Failed check = WARNING. State transition = WARNING. Recovery = INFO. Suppress repeated identical failures. Full detail at DEBUG.

8. **Remove old `_probe_once()` and `_fail_count`** — superseded by `_verify_all()` and immediate local failure detection.

Important: The `_run_cmd`, `_verify_listener`, `_verify_routing`, `_verify_upstream` methods remain as instance methods (not a separate verifier class) — Claude's discretion per CONTEXT.md. Tests will mock these individual methods.

```python
async def handle_get_proxy_status(params):
    verify = False
    if isinstance(params, dict):
        v = params.get("verify", False)
        if not isinstance(v, bool):
            return {"error": "verify must be a boolean"}
        verify = v
    return await proxy.get_status(verify=verify)
```

**Note:** The existing request parsing in `parse_set_proxy_route_request` (line 31) is strict about boolean types. The `verify` parameter must follow the same discipline — reject truthy junk like `"false"` or `1`.

This returns the full response shape: `{state, checks, error_code, error, verified_at, live_check}`.

The C++ SystemServiceClient already reads `state` and `error` from the result object (line 153-167 of SystemServiceClient.cpp), so it will continue working — the new fields (`checks`, `error_code`, `verified_at`, `live_check`) are additive and backward-compatible. No C++ changes needed.

Also update `handle_set_proxy_route` to use `get_status()` for the response after enable/disable, so the response includes the full verified status shape instead of just `{ok, state}`. **Both the success path AND the exception path must return the full response shape.** The exception handler currently returns `{ok, error, state}` which drops checks, error_code, verified_at, and live_check — exactly the optimistic reporting this phase is fixing.

```python
async def handle_set_proxy_route(params):
    try:
        normalized = parse_set_proxy_route_request(params)
        if normalized["active"]:
            await proxy.enable(...)
        else:
            await proxy.disable()
        status = await proxy.get_status()
        ok = status["state"] != "failed"
        return {"ok": ok, **status}
    except Exception as e:
        LOG.error("set_proxy_route error: %s", e)
        # Still return full shape — get_status() reflects the post-failure state
        try:
            status = await proxy.get_status()
        except Exception:
            status = {"state": proxy.state.value, "checks": {"listener": False, "iptables": False, "upstream": False}, "error_code": "listener_down", "error": str(e), "verified_at": None, "live_check": False}
        return {"ok": False, "error": str(e), **status}
```
