<objective>
Add `redsocks` system user creation to both installers so the owner-based iptables exemption works on hardware.

Purpose: The ProxyManager's owner-based exemption (`-m owner --uid-owner redsocks`) requires the `redsocks` system user to exist on the Pi. This user is a locked-down system account (no login, no home) — it exists solely so redsocks traffic can be identified and exempted from the redirect chain.

Output: Both installers create the redsocks user alongside the existing openauto group setup.
</objective>

  <files>install.sh</files>
  <action>
In the `create_system_service()` function, after the openauto group creation block (the section that does `groupadd openauto` and `usermod -aG openauto`), add redsocks system user creation:

```bash
# Create dedicated redsocks system user for transparent proxy owner-based exemption
if ! id -u redsocks &>/dev/null; then
    info "Creating redsocks system user..."
    sudo useradd --system --no-create-home --shell /usr/sbin/nologin redsocks
    info "Created redsocks system user"
else
    info "redsocks system user already exists"
fi
```

This is idempotent — `id -u redsocks` check prevents duplicate creation on re-runs. The `--system` flag gives it a low UID, `--no-create-home` skips home directory, `--shell /usr/sbin/nologin` prevents login.

Place this right after the openauto group block, before the service template rendering section.
