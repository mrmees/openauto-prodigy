  <how-to-verify>
    **Matt does:**
    1. On the Pi, run the installer (or ensure latest code is deployed): `cd ~/openauto-prodigy && git pull && bash install.sh`
    2. Reboot or log out and back in (for openauto group membership to take effect)
    3. Launch Prodigy (or let systemd auto-start it)

    **Claude verifies over SSH (after Matt confirms installer ran):**
    ```bash
    # 1. Verify openauto-system is active and running as root
    ssh matt@192.168.1.152 'systemctl is-active openauto-system && ps -o user= -p $(pgrep -f openauto_system.py)'
    # Expected: "active" and "root"

    # 2. Verify socket exists with correct permissions
    ssh matt@192.168.1.152 'stat -c "%U:%G %a" /run/openauto/system.sock'
    # Expected: "root:openauto 660"

    # 3. Verify no permission-denied errors in recent journal
    ssh matt@192.168.1.152 'journalctl -u openauto-system --since "5 minutes ago" --no-pager | grep -i "permission denied" || echo "NO PERMISSION ERRORS"'
    # Expected: "NO PERMISSION ERRORS"

    # 4. Exercise IPC auth — verify authorized client can call a method
    ssh matt@192.168.1.152 'echo "{\"method\":\"get_health\",\"params\":{}}" | socat - UNIX-CONNECT:/run/openauto/system.sock'
    # Expected: JSON response with health data (not "permission denied")
    # This proves the peer credential auth accepts the matt user (in openauto group).
    # If socat is not installed, use: python3 -c "import socket,json; s=socket.socket(socket.AF_UNIX); s.connect('/run/openauto/system.sock'); s.send(json.dumps({'method':'get_health','params':{}}).encode()+'\\n'.encode()); print(s.recv(4096).decode())"
    ```

    **Matt confirms locally:**
    - Prodigy UI launched without socket/permission errors
    - No error dialogs or connection failures visible

    **Interpretation boundary:**
    - Socket/client failure = Phase 15 regression → stop, fix Phase 15, rerun this checkpoint
    - Socket clean + routing fails later = Phase 16 territory → proceed
