  <name>Task 2: Implement frameSwapped splash dismissal in main.cpp</name>
  <files>src/main.cpp</files>
  <action>
Add a one-shot QQuickWindow::frameSwapped handler in main.cpp that kills the swaybg splash process after the first rendered frame.

After the existing block that wires DisplayInfo to the root window (around line 960-1000), add the splash dismissal logic:

```cpp
// --- Splash dismissal: kill swaybg after first frame is visible ---
{
    auto* rootWindow = qobject_cast<QQuickWindow*>(engine.rootObjects().first());
    if (rootWindow) {
        QObject::connect(rootWindow, &QQuickWindow::frameSwapped, &app,
            [rootWindow, &app]() {
                // First frame presented to compositor -- dismiss splash
                QTimer::singleShot(500, &app, []() {
                    QProcess::execute("pkill", {"-f", "swaybg.*splash"});
                });
                // One-shot: disconnect after first fire
                rootWindow->disconnect(&app);
            });
    }
}
```

Key implementation details per ARCHITECTURE.md and PITFALLS.md:
- Use `QQuickWindow::frameSwapped` signal (not afterRendering -- too early)
- 500ms delay via QTimer::singleShot ensures compositor has presented the frame (Pitfall 6)
- `pkill -f "swaybg.*splash"` pattern matches only the kiosk splash instance, not any desktop swaybg (ARCHITECTURE.md pattern)
- One-shot via disconnect after first fire
- Use QProcess::execute (synchronous, brief) for the pkill call
- Add `#include <QProcess>` if not already present

Also add the QProcess include near the top of the file.
