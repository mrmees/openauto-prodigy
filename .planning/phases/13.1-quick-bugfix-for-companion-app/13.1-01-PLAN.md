**Implementation changes** (`CompanionListenerService.cpp`):

1. **Constructor:** Create `inactivityTimer_` as member QTimer with `setSingleShot(true)`, `setInterval(30000)`. Connect `timeout` signal to lambda that logs warning with peer info and calls `clearClientSession()`.

2. **`setInactivityTimeout(int ms)`:** Sets `inactivityTimeoutMs_` and calls `inactivityTimer_->setInterval(ms)`.

3. **`clearClientSession()`:** Extract from current `onClientDisconnected()` body. Implementation per RESEARCH.md pattern:
   - Guard: `if (!client_) return;`
   - `client_->disconnect(this);` — MUST come before abort to prevent re-entrant signal emission
   - `client_->abort();` + `client_->deleteLater();` + `client_ = nullptr;`
   - Clear: `sessionKey_.clear()`, `lastSeq_ = -1`
   - Stop: `inactivityTimer_->stop()`
   - Clear all published state (GPS, battery, internet, proxy route, theme transfer)
   - Call `systemClient_->setProxyRoute(false)` if systemClient_ non-null
   - Emit: `connectedChanged()`, `gpsChanged()`, `batteryChanged()`, `internetChanged()`

4. **`onNewConnection()`:** Replace the current reject-if-connected logic:
   - Get `pending = server_->nextPendingConnection()`
   - If `client_` non-null: log warning with old peer + new peer + "reason=replaced_by_new_connection", then call `clearClientSession()`
   - Set `client_ = pending`
   - Connect `readyRead`, `disconnected` (existing), AND `errorOccurred` (new — lambda logs error string then calls `clearClientSession()`)
   - Start `inactivityTimer_` (start/restart 30s countdown)
   - Call existing `sendChallenge()`

5. **`onClientDisconnected()`:** Replace body with: `qCInfo(lcCore) << "Companion: client disconnected"; clearClientSession();`

6. **`handleStatus()`:** After successful MAC verification (after `verifyMac` returns true), add `inactivityTimer_->start();` to reset the 30s countdown.

**Critical implementation notes per user decisions:**
- Cleanup MUST be idempotent — errorOccurred followed by disconnected must not double-free
- Log old peer address/port, new peer address/port, and reason on replacement path
- Timer resets only on valid MAC-verified status messages, not on hello or theme messages
- Use `abort()` not `disconnectFromHost()` for immediate teardown
