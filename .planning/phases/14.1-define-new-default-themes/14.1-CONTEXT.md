# Phase 14.1: Define New Default Themes - Context

**Gathered:** 2026-03-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Promote 9 visually-approved themes from the Pi's user theme directory into the repo as bundled defaults, replacing the 4 old bundled themes. The 9 themes already exist on the Pi (`~/.openauto/themes/`), were created via the companion app, and have been approved on hardware. This phase is a migration — pull YAMLs and wallpaper JPEGs from the Pi, rename prodigy→default, commit into `config/themes/`, and remove the old set. No theme creation or color exploration needed. Keep the connected-device internal theme.

</domain>

### Default theme
- Prodigy becomes the new `default` theme (id: default, name: Prodigy or similar)

- Include wallpaper JPEGs from each Pi theme in the bundled theme directory
- Themes should look complete out of the box without companion app

- Use companion-app-generated M3 palettes as-is — no manual color tweaking

### Scope note
- This intentionally supersedes the PROJECT.md out-of-scope note "Bundled theme YAML changes — Themes built via companion app and imported, not shipped in binary." The themes were built via companion app as intended; promoting them to bundled is a natural next step now that the lineup is finalized.
