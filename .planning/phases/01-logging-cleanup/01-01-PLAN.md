Create `src/core/Logging.hpp`:
- `Q_DECLARE_LOGGING_CATEGORY` for lcAA ("oap.aa"), lcBT ("oap.bt"), lcAudio ("oap.audio"), lcPlugin ("oap.plugin"), lcUI ("oap.ui"), lcCore ("oap.core")
- Declare namespace oap functions: `installLogHandler()`, `setVerbose(bool)`, `setDebugCategories(const QStringList&)`, `isVerbose()`

Create `src/core/Logging.cpp`:
- `Q_LOGGING_CATEGORY` definitions with `QtInfoMsg` as third arg (quiet by default)
- `static std::atomic<bool> g_verbose{false}` for thread-safe verbose flag
- `static bool g_logToFile{false}` and `static FILE* g_logFile{nullptr}` for optional file output
- Custom message handler (`oap::logMessageHandler`):
  - Null-check `ctx.category` (use "default" if null — Pitfall 5 from research)
  - Format: `[HH:mm:ss.zzz] [CATEGORY] message` in quiet mode
  - Format: `[HH:mm:ss.zzz] [CATEGORY] [threadName] message` in verbose mode
  - Use short bracket tags: [AA], [BT], [Audio], [Plugin], [UI], [Core], [Lib], [App]
  - Library detection: check `ctx.file` for "open-androidauto" path OR `ctx.category` starts with "oaa." OR message matches known bracket tags ([TCPTransport], [ControlChannel], etc.)
  - In quiet mode: suppress library debug/info UNLESS message matches lifecycle patterns (contains "opened", "closed", "connected", "disconnected", "session")
  - Thread-safe: use `fprintf(stderr, ...)` (OS serializes stderr writes)
  - If `g_logToFile`, also write to file
- `setVerbose()`: set `g_verbose`, call `QLoggingCategory::setFilterRules()` with complete ruleset (Pitfall 2: replaces ALL rules)
- `setDebugCategories()`: build complete filter ruleset enabling only specified categories; if "aa" is in list, also enable "oaa.*"
- `installLogHandler()`: call `qInstallMessageHandler(oap::logMessageHandler)`

Add `Logging.cpp` to `openauto-core` STATIC library sources in `src/CMakeLists.txt`.
