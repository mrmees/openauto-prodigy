<task type="auto">
  <name>Task 1: Harden Wallpaper.qml — sourceSize, clip, retainWhileLoading</name>
  <files>qml/components/Wallpaper.qml</files>
  <action>
Update Wallpaper.qml with three hardening changes:

1. **Add `clip: true`** to the root Item — prevents PreserveAspectCrop from painting outside the component bounds into adjacent regions (navbar, widget area).

2. **Add `sourceSize`** to the Image element, bound to full shell dimensions:
   ```qml
   sourceSize.width: DisplayInfo.windowWidth
   sourceSize.height: DisplayInfo.windowHeight
   ```
   This caps the decoded texture to display dimensions regardless of the source file resolution (e.g., an 8MP JPEG only allocates a 1024x600 texture). The original file stays full resolution on disk.

3. **Add `retainWhileLoading: true`** to the Image element — Qt 6.8 feature that holds the previous image visible while the new source loads asynchronously. This prevents the flash-to-background-color that occurs when `source` changes (theme switch, day/night toggle). The existing `asynchronous: true` is already set.

Do NOT add a subpixel margin to sourceSize (per Claude's discretion — not worth it for this use case).

Keep the existing Background Rectangle with its 300ms ColorAnimation for themes with no wallpaper.
  </action>

  </verify>
