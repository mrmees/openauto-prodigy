**Create src/core/services/WeatherService.cpp:**

Implement all methods. Key details:

1. `roundCoordinate(lat, lon)`: Round each to 2 decimal places using `QString::number(val, 'f', 2)`. Return `"lat:lon"` string as cache key.

2. `buildUrl(lat, lon)`: Return:
   ```
   https://api.open-meteo.com/v1/forecast?latitude=LAT&longitude=LON&current=temperature_2m,relative_humidity_2m,apparent_temperature,weather_code,wind_speed_10m,wind_direction_10m,is_day&timezone=auto
   ```
   Use `QString::arg` with 4 decimal precision for lat/lon.

3. `getWeatherData(lat, lon)`:
   - Compute key via `roundCoordinate`
   - If key exists in `cache_`, return existing `WeatherData*`
   - Otherwise create new `WeatherData`, store in cache and `coordsByKey_`, call `fetchWeather()`, call `fetchLocationName()`
   - Call `cleanupStaleEntries()` if cache exceeds `MAX_CACHE_SIZE`
   - Return the WeatherData pointer

4. `subscribe(lat, lon, intervalMinutes)`:
   - Compute key via `roundCoordinate`
   - If key not in cache, call `getWeatherData(lat, lon)` first to create entry
   - Convert `intervalMinutes` to ms. Append `intervalMs` to `subscriberIntervals_[key]` list.
   - Effective interval for this location = `*std::min_element(subscriberIntervals_[key])` (shortest wins)
   - If data is stale (lastUpdated invalid OR age > effective interval), call `fetchWeather()` immediately
   - This is how "resume + immediate fetch on page return if data is stale" works

4a. `triggerRefreshTimer()`: Public test seam — calls `onRefreshTimer()`.
4b. `triggerCleanup()`: Public test seam — calls `cleanupStaleEntries()`.

5. `unsubscribe(lat, lon, intervalMinutes)`:
   - Compute key via `roundCoordinate`
   - Convert `intervalMinutes` to ms. Remove ONE matching entry from `subscriberIntervals_[key]` list (use `removeOne(intervalMs)` — removes first match only, handles duplicate intervals correctly)
   - If list is now empty, remove the key from `subscriberIntervals_` entirely (no subscribers = no interval)
   - Effective interval recalculates naturally on next timer tick via `*std::min_element`
   - Do NOT delete the cache entry here — let cleanupStaleEntries handle it

6. `subscriberCount(key) const`: Return `subscriberIntervals_.value(key).size()`. (Exposed for testing.)
6a. `effectiveIntervalMs(key) const`: Return `subscriberIntervals_[key].isEmpty() ? 0 : *std::min_element(...)`. (Exposed for testing.)

7. `fetchWeather(key, lat, lon)`:
   - Set data loading=true
   - `QNetworkRequest` to `buildUrl(lat, lon)`
   - Connect `QNetworkReply::finished` to `onWeatherReplyFinished`

8. `onWeatherReplyFinished(reply, data)`:
   - Check reply error. On error: `data->setError(reply->errorString())`, `data->setLoading(false)`
   - On success: call `parseResponse(data, reply->readAll())`
   - `reply->deleteLater()`

9. `parseResponse(data, jsonData)`:
   - Parse with `QJsonDocument::fromJson`
   - Extract `obj["current"]` as QJsonObject
   - If missing "current" key: `data->setError("Invalid response: missing current data")`
   - Otherwise call `data->updateFromJson(currentObj)` and `data->setLoading(false)`

10. `WeatherData::updateFromJson(currentObj)`:
   - `tempC_ = currentObj["temperature_2m"].toDouble()`
   - `feelsLikeC_ = currentObj["apparent_temperature"].toDouble()`
   - `weatherCode_ = currentObj["weather_code"].toInt()`
   - `humidity_ = currentObj["relative_humidity_2m"].toDouble()`
   - `windSpeedKph_ = currentObj["wind_speed_10m"].toDouble()`
   - `windDirection_ = currentObj["wind_direction_10m"].toInt()`
   - `isDay_ = currentObj["is_day"].toInt() == 1`
   - `lastUpdated_ = QDateTime::currentDateTime()`
   - `error_ = ""`
   - `loading_ = false`
   - emit `dataChanged()`

11. `fetchLocationName(data, lat, lon)`: Reverse geocoding call to:
   ```
   https://geocoding-api.open-meteo.com/v1/reverse?latitude=LAT&longitude=LON&count=1
   ```
   On success, parse `results[0].name` and call `data->setLocationName(name)`.
   On failure, set locationName to empty string (widget shows coordinates or nothing).

12. `requestRefresh(lat, lon)`: Look up key, if exists call `fetchWeather()`.

13. `onRefreshTimer()`: Iterate all `coordsByKey_` entries. **Only call `fetchWeather()` for entries where `subscriberIntervals_[key]` is non-empty AND data is older than `effectiveIntervalMs(key)`.** Skip entries with no subscribers — this is how "pause when page hidden" works at the service level. The per-location effective interval (shortest among active subscribers) means a widget configured for 60-minute refresh won't fetch every 5 minutes. Timer runs at a fixed 1-minute tick to check all locations against their individual intervals.

14. `cleanupStaleEntries()`: If `cache_.size() > MAX_CACHE_SIZE`, find entries with the oldest `lastUpdated` **that have no subscribers** (`subscriberIntervals_[key].isEmpty()`). Only evict entries with no active subscribers. If all entries have active subscribers, do NOT evict (this prevents invalidating live QML objects). Use `deleteLater()` for safe cleanup. Also remove from `coordsByKey_` and `subscriberIntervals_`.

15. Constructor: Create `nam_` (new QNetworkAccessManager), create `refreshTimer_` with 60000ms (1-minute tick — each tick checks per-location intervals), connect to `onRefreshTimer`, start timer.

**Part A: Create qml/widgets/WeatherWidget.qml**

Follow the BatteryWidget.qml pattern for null-safe CompanionService access. Key structure:

```qml
import QtQuick
import QtQuick.Layouts

Item {
    id: weatherWidget
    clip: true

    property QtObject widgetContext: null
    readonly property int colSpan: widgetContext ? widgetContext.colSpan : 1
    readonly property int rowSpan: widgetContext ? widgetContext.rowSpan : 1
    readonly property bool isCurrentPage: widgetContext ? widgetContext.isCurrentPage : true

    // Per-instance config
    readonly property var currentEffectiveConfig: widgetContext ? widgetContext.effectiveConfig : ({})
    readonly property string tempUnit: currentEffectiveConfig.unit || "fahrenheit"
    readonly property string refreshInterval: currentEffectiveConfig.refresh || "5"

    // Breakpoints
    readonly property bool showIcon: colSpan >= 2 || rowSpan >= 2
    readonly property bool showExtended: colSpan >= 3 && rowSpan >= 3

    // GPS + companion state (null-safe)
    // IMPORTANT: Use gpsStale (not gpsLat !== 0) to detect GPS availability.
    // gpsLat === 0 rejects valid equator coordinates. gpsStale is the proper
    // signal from CompanionListenerService for "GPS data is available and current".
    readonly property bool companionConnected: CompanionService ? CompanionService.connected : false
    readonly property bool hasGps: companionConnected
                                   && CompanionService
                                   && !CompanionService.gpsStale
    readonly property double gpsLat: CompanionService ? CompanionService.gpsLat : 0
    readonly property double gpsLon: CompanionService ? CompanionService.gpsLon : 0

    // Weather data object (requested imperatively, not in binding)
    property QtObject weatherData: null

    // Track previous subscription to unsubscribe on location change
    property double subscribedLat: NaN
    property double subscribedLon: NaN

    // Temperature conversion
    function displayTemp(tempC) {
        if (tempUnit === "celsius") return Math.round(tempC) + "\u00B0C"
        return Math.round(tempC * 9/5 + 32) + "\u00B0F"
    }

    // WMO weather code to MaterialIcon mapping
    function weatherIcon(code, isDay) {
        if (code === 0) return isDay ? "\ue81a" : "\uf159"  // wb_sunny / dark_mode
        if (code <= 2) return isDay ? "\ue42d" : "\uf172"   // wb_cloudy / nights_stay
        if (code === 3) return "\ue42d"                       // cloud
        if (code === 45 || code === 48) return "\ue818"       // foggy
        if (code >= 51 && code <= 67) return "\uf176"         // rainy
```

**Subscribe/unsubscribe lifecycle:** The widget calls `WeatherService.subscribe(lat, lon)` when it should be actively receiving updates (page visible + GPS available), and `WeatherService.unsubscribe(lat, lon)` when it should not. This controls which locations the service-side timer refreshes.

```qml
// Track the interval we subscribed with (needed for unsubscribe)
property int subscribedInterval: 5

// Unsubscribe from previous location if any
function doUnsubscribe() {
    if (!isNaN(subscribedLat) && !isNaN(subscribedLon) && WeatherService) {
        WeatherService.unsubscribe(subscribedLat, subscribedLon, subscribedInterval)
        subscribedLat = NaN
        subscribedLon = NaN
    }
}

// Subscribe to current location — passes per-instance refresh interval
function doSubscribe() {
    if (hasGps && isCurrentPage && WeatherService) {
        doUnsubscribe()  // unsub previous if location changed
        var interval = parseInt(refreshInterval)
        weatherData = WeatherService.getWeatherData(gpsLat, gpsLon)
        WeatherService.subscribe(gpsLat, gpsLon, interval)
        subscribedLat = gpsLat
        subscribedLon = gpsLon
        subscribedInterval = interval
    }
}

function updateWeatherData() {
    if (hasGps && isCurrentPage && WeatherService) {
        doSubscribe()
    } else {
        doUnsubscribe()
        weatherData = null
    }
}

onHasGpsChanged: updateWeatherData()
onGpsLatChanged: updateWeatherData()
onGpsLonChanged: updateWeatherData()

Component.onCompleted: updateWeatherData()
Component.onDestruction: doUnsubscribe()
```
