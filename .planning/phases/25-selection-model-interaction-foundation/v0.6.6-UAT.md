## Tests

### 1. Long-Press Widget Selection
expected: Long-press (hold ~500ms) on any widget. The widget should visually lift (slight scale up + drop shadow behind it). Other widgets stay completely unchanged.
result: pass

### 2. Release Without Drag = Selected State
expected: Long-press a widget, then release without moving your finger. The widget should show a primary-colored accent border. Dotted grid lines should appear across the entire page. The widget returns to normal scale (no lift, just border).
result: pass

### 3. Drag to Reposition
expected: Long-press a widget and drag it. It should follow your finger at reduced opacity with a placeholder at the original position. Release over a valid grid cell — widget snaps into place. Release over an occupied cell — widget snaps back to original position.
result: pass (fixed — interceptor-owned drag, immediate on selected widget)

### 4. Deselect via Tap Empty Space
expected: With a widget selected (accent border visible), tap on empty grid space. The selection should clear — accent border disappears, grid lines disappear, navbar returns to normal.
result: pass

### 5. Deselect via Tap Another Widget
expected: With a widget selected, tap a different widget. The selection should clear (deselect only — the tapped widget's normal action should NOT fire). A second tap on that widget should then fire its action normally.
result: pass

### 6. Auto-Deselect Timeout
expected: Select a widget (long-press and release). Wait ~10 seconds without touching anything. The selection should automatically clear — accent border gone, grid lines gone, navbar reverted.
result: pass

### 7. SwipeView Locked During Selection
expected: With a widget selected, try to swipe left/right to change pages. Swiping should NOT work — the page should stay locked. Deselect the widget, then swiping should work again.
result: pass

### 8. Interactive Widget Long-Press
expected: Long-press on an interactive widget (e.g., Now Playing skip button, AA Focus toggle, Weather widget). The widget should lift and become selected, NOT fire the widget's tap action. After deselecting, tapping the widget's controls should work normally.
result: pass

### 9. Navbar Gear/Trash Transformation
expected: Select a widget. The navbar volume control should instantly become a gear (settings) icon, and the brightness control should become a trash icon. The center clock should show the selected widget's display name instead of the time.
result: pass

### 10. Navbar Gear Opens Config Sheet
expected: Select a widget that has configuration options (e.g., Clock with style selection). Tap the gear icon on the navbar. The widget's config sheet should open. Close it and verify the widget is still selected.
result: pass

### 11. Navbar Trash Deletes Widget
expected: Select a non-singleton widget. Tap the trash icon on the navbar. The widget should be removed from the grid. Navbar should revert to volume/brightness.
result: pass

### 12. Navbar Gear/Trash Dimming
expected: Select a widget with no configuration (e.g., a simple display widget). The gear icon should appear dimmed/disabled — tapping it should do nothing. Select a singleton widget (e.g., launcher). The trash icon should appear dimmed — tapping it should do nothing.
result: pass

### 13. Navbar Reverts on Drag
expected: Select a widget (navbar shows gear/trash). Start dragging the widget. The navbar should revert to volume/brightness while dragging. Release the widget — if it's still selected, navbar should show gear/trash again.
result: pass

### 14. Edge Resize Handles Visible
expected: Select a widget. Four pill-shaped resize handles should appear centered on each edge (top, bottom, left, right) of the selected widget.
result: pass

### 15. Edge Resize — Right/Bottom (Grow)
expected: Select a widget. Drag the right edge handle to the right. A ghost rectangle should preview the new size (green border = valid). Release — widget resizes. Same with the bottom edge to grow downward.
result: pass

### 16. Edge Resize — Left/Top (Move + Grow)
expected: Select a widget. Drag the left edge handle to the left. The widget should grow leftward — the right edge stays anchored. Ghost rectangle previews the new position+size. Same with the top edge to grow upward.
result: pass

### 17. Edge Resize Min/Max Constraints
expected: Try to resize a widget smaller than its minimum size or larger than its maximum. The resize should clamp, and a brief color flash should appear on the ghost rectangle when hitting the limit.
result: pass

### 18. Edge Resize Collision
expected: Try to resize a widget so it overlaps another widget. The ghost rectangle should turn red (invalid). Releasing should NOT apply the resize — widget stays at original size.
result: pass

### 19. Trash Deletes Last Widget on Page → Page Auto-Deleted
expected: Navigate to a page with only one widget. Select it and tap the trash icon. The widget should be removed AND the now-empty page should be automatically deleted. You should be navigated to the previous page.
