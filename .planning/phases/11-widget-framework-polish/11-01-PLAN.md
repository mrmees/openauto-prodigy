    2. In HomeMenu.qml's widget delegate (around line 229), create a WidgetInstanceContext per delegate:
       ```qml

       WidgetInstanceContext {
           id: widgetCtx
           // Note: WidgetInstanceContext constructor needs GridPlacement, cellWidth, cellHeight, IHostContext
           // Since QML can't call C++ constructors with custom types, use a different approach:

       ```

       ACTUALLY -- WidgetInstanceContext takes a GridPlacement struct and IHostContext* in its constructor, which QML cannot provide. The cleaner approach is:

       a. Add a factory method to WidgetGridModel (or a standalone helper) that creates a WidgetInstanceContext from a model index. Something like:
          `Q_INVOKABLE QObject* createWidgetContext(const QString& instanceId, QObject* parent = nullptr)`
          This creates a WidgetInstanceContext with the correct placement, cellWidth (from gridCols_), cellHeight, and hostContext.

       b. OR (simpler, per research recommendation): Create properties directly on the Loader and use a lightweight QML QtObject for context. Since WidgetInstanceContext already exists in C++ with provider access, expose it via a `Q_INVOKABLE` factory on WidgetGridModel.

       Recommended approach:
       - Add `Q_INVOKABLE QObject* createWidgetContext(const QString& instanceId, QObject* parent)` to WidgetGridModel
       - In HomeMenu.qml delegate, after Loader loads:
         ```qml
         property var widgetCtx: null
         Component.onCompleted: {
             widgetCtx = WidgetGridModel.createWidgetContext(model.instanceId, widgetLoader)
         }
         ```
       - Set widgetCtx as a context property on the Loader's item:
         ```qml
         Loader {
             onLoaded: {
                 if (item) item.widgetContext = Qt.binding(function() { return delegateItem.widgetCtx })
                 // OR set as context property via C++ approach
             }
         }
         ```
       - Actually, the simplest reliable approach: add properties to the Loader itself that widgets read via `parent.colSpan` etc. BUT the CONTEXT.md prescribes widgetContext.

       FINAL approach (simplest that honors CONTEXT.md):
       - Add `Q_INVOKABLE QObject* createWidgetContext(const QString& instanceId, QObject* parent)` to WidgetGridModel.hpp/.cpp
       - In HomeMenu.qml delegate, create context and bind spans:
         ```qml
         property QtObject widgetCtx: WidgetGridModel.createWidgetContext(model.instanceId, delegateItem)
         Connections {
             target: widgetCtx
             // Keep spans synced when model changes (resize)
         }
         // Bind spans:
         Binding { target: widgetCtx; property: "colSpan"; value: model.colSpan }
         Binding { target: widgetCtx; property: "rowSpan"; value: model.rowSpan }
         ```
       - In the Loader, forward to loaded widget via a property:
         The loaded widget QML files will declare `property QtObject widgetContext: null` and the delegate sets it:
         ```qml
         Loader {
             onLoaded: {
                 if (item && "widgetContext" in item) item.widgetContext = widgetCtx
             }
         }
         ```

       QObject* WidgetGridModel::createWidgetContext(const QString& instanceId, QObject* parent) {
