WidgetInstanceContext (after Plan 01):

Widget QML access pattern:
```qml
// Widget declares a property to receive context from HomeMenu delegate
property QtObject widgetContext: null

// Span-based breakpoints
readonly property int colSpan: widgetContext ? widgetContext.colSpan : 1
readonly property int rowSpan: widgetContext ? widgetContext.rowSpan : 1
readonly property bool showDate: colSpan >= 2

// Provider access
property bool connected: widgetContext && widgetContext.projectionStatus
                         ? widgetContext.projectionStatus.projectionState === 3 ... : false
```

Span-to-pixel mapping (from RESEARCH.md):
| Widget | Old Pixel Check | New Span Check |
|--------|----------------|----------------|
| ClockWidget | width >= 250 | colSpan >= 2 |
| ClockWidget | height >= 180 | rowSpan >= 2 |
| AAStatusWidget | width >= 250 | colSpan >= 2 |
| NowPlayingWidget | width >= 400 | colSpan >= 3 |
| NowPlayingWidget | height >= 180 | rowSpan >= 2 |
| NavigationWidget | width >= 400 | colSpan >= 3 |
| NavigationWidget | height >= 180 | rowSpan >= 2 |
| NavigationWidget | width >= 500 | colSpan >= 4 |

Placeholder state pattern (formalized):
```qml
Item {
    anchors.centerIn: parent
    visible: !dataAvailable
    opacity: 0.4
    ColumnLayout {
        anchors.centerIn: parent
        spacing: UiMetrics.spacing
        MaterialIcon {
            icon: "\ue405"  // widget-specific
            size: rowSpan >= 2 ? UiMetrics.iconSize * 2 : UiMetrics.iconSize * 1.5
            color: ThemeService.onSurfaceVariant
            Layout.alignment: Qt.AlignHCenter
        }
        NormalText {
            text: "No data"  // widget-specific
            visible: colSpan >= 2 || rowSpan >= 2
            font.pixelSize: UiMetrics.fontBody
            color: ThemeService.onSurfaceVariant
            Layout.alignment: Qt.AlignHCenter
        }
    }
}
```

pressAndHold forwarding (required on ALL widgets):
```qml
MouseArea {
    anchors.fill: parent
    z: -1  // behind interactive controls
    pressAndHoldInterval: 500
    onPressAndHold: {
        if (root.parent && root.parent.requestContextMenu)
            root.parent.requestContextMenu()
    }
}
```
Interactive widgets (NowPlaying, Navigation, AAStatus) with their own tap MouseAreas must ALSO forward pressAndHold from those areas.
