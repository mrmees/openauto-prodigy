Create `qml/widgets/AAFocusToggleWidget.qml` â€” a toggle widget that switches AA between projected and native (backgrounded) states.

**State reading:**
```qml
readonly property int projState: widgetContext && widgetContext.projectionStatus
    ? widgetContext.projectionStatus.projectionState : 0
readonly property bool aaConnected: projState >= 3  // Connected(3) or Backgrounded(4)
readonly property bool isProjected: projState === 3  // Connected = projected
readonly property bool isBackgrounded: projState === 4  // Backgrounded = native/car mode
```

**Visual states:**
- When `isProjected` (Connected, state 3): phone icon `"\ue325"` (smartphone) + "AA" label, full opacity, indicates phone is being projected
- When `isBackgrounded` (state 4): car icon `"\ueff7"` (directions_car) + "Car" label, full opacity, indicates car UI is active
- When AA not connected (`projState < 3`): car icon `"\ueff7"` greyed out, dimmed opacity (0.4), "--" or "Off" label, no tap response

**Layout:** Centered ColumnLayout matching other 1x1 widgets:
- MaterialIcon with state-dependent icon and color
- NormalText label below

**Tap action:**
```qml
onClicked: {
    if (aaFocusWidget.isProjected)
        ActionRegistry.dispatch("aa.exitToCar")
    else if (aaFocusWidget.isBackgrounded)
        ActionRegistry.dispatch("aa.requestFocus")
    // else: not connected, ignore tap
}
```

**Colors:**
- Projected: `ThemeService.primary` icon color (active, phone showing)
- Backgrounded: `ThemeService.onSurface` icon color (car mode active)
- Not connected: `ThemeService.onSurfaceVariant` with 0.4 opacity on entire widget

Follow established widget patterns: `property QtObject widgetContext: null`, colSpan/rowSpan bindings, `MouseArea z:-1` for pressAndHold context menu (separate from click area, or combined with click handler), all sizes via `UiMetrics`, all colors via `ThemeService`.
