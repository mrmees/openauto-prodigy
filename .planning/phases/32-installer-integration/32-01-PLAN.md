# Plan: Installer Integration for Kiosk Session & Boot Splash

## Objective

Wrap all validated Phase 28-31 work (kiosk session config, compositor splash, boot splash, exit-to-desktop) into install.sh as two new functions: `create_kiosk_session()` and `configure_boot_splash()`. Replace the existing AUTOSTART prompt with a kiosk mode prompt. Handle idempotent upgrades.

## Context

- @docs/pi-config/kiosk/ -- Reference config files created in Phase 28
- @docs/pi-config/kiosk/boot-splash.md -- Boot splash deployment steps from Phase 30
- @docs/pi-config/kiosk/labwc/autostart -- Compositor splash from Phase 29
- @.planning/research/ARCHITECTURE.md -- Installer integration section (Pattern 8)
- @.planning/research/PITFALLS.md -- Pitfalls 2, 7, 12, 13, 16
- @install.sh -- Current installer

## Tasks

### Task 1: Add kiosk session creation function
type="auto"

Add `create_kiosk_session()` function to install.sh that deploys:
- XDG session file (`openauto-kiosk.desktop`) to `/usr/share/wayland-sessions/`
- Kiosk labwc config directory (`/etc/openauto-kiosk/labwc/`) with rc.xml, autostart, environment
- LightDM drop-in (`50-openauto-kiosk.conf`) to `/etc/lightdm/lightdm.conf.d/`
- AccountsService session entry to `/var/lib/AccountsService/users/$USER`
- Splash image (`assets/splash.png`) to `/usr/share/openauto-prodigy/`

All files are written via heredocs (matching reference configs in `docs/pi-config/kiosk/`), not copied from the repo. The function must be idempotent -- safe to re-run on upgrade.

**Done when:** Function exists, writes all files, uses `$USER` variable (not hardcoded), outputs TUI status messages via `ok`/`run_with_spinner`.

### Task 2: Add boot splash configuration function
type="auto"

Add `configure_boot_splash()` function to install.sh that:
1. Installs `rpi-splash-screen-support` package (if available, non-fatal if not)
2. Copies pre-built TGA (`assets/splash.tga`) to `/lib/firmware/logo.tga`
3. Runs `configure-splash /lib/firmware/logo.tga` (with fallback on failure)
4. Repairs cmdline.txt (re-adds `quiet loglevel=0 logo.nologo` stripped by configure-splash, removes `splash` and `plymouth.ignore-serial-consoles`)
5. Masks Plymouth services (plymouth-start, plymouth-quit, plymouth-quit-wait)
6. Runs `update-initramfs -u`
7. Verifies: hook exists, TGA in initramfs, cmdline.txt correct

Boot splash is nice-to-have. All steps are wrapped in error handling -- failures log warnings and continue. The function follows `docs/pi-config/kiosk/boot-splash.md` step-by-step.

Add `rpi-splash-screen-support` to the PACKAGES array in `install_dependencies()`.

**Done when:** Function exists with full error handling, non-fatal failures, and verification checks.

### Task 3: Replace AUTOSTART prompt with kiosk mode prompt and wire into main flow
type="auto"

In `setup_hardware()`:
- Replace the "Start OpenAuto Prodigy automatically on boot?" prompt with "Enable kiosk mode? (recommended)" prompt
- Set `KIOSK_MODE=true/false` instead of `AUTOSTART` (keep `AUTOSTART=true` when kiosk is enabled since kiosk implies autostart)

In `main()`:
- After the Services step (step 6), add a new conditional block:
  - If `KIOSK_MODE=true`: call `create_kiosk_session()` then `configure_boot_splash()`
  - If `KIOSK_MODE=false`: skip kiosk setup, keep desktop-mode service
- Update the step registry to add a "Kiosk" step (renumber as needed)

In `finalize()`:
- Add kiosk mode status to the final summary
- When kiosk mode is enabled, show revert instructions: "To switch back to desktop: sudo rm /etc/lightdm/lightdm.conf.d/50-openauto-kiosk.conf && sudo reboot"

**Done when:** Full flow works -- kiosk prompt appears, yes enables kiosk+splash, no skips them. Finalize shows appropriate summary.
