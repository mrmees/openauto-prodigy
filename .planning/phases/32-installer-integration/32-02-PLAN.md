# Plan 32-02: Prebuilt Installer Kiosk Integration

## Objective

Add kiosk session creation, boot splash configuration, and LightDM autologin to `install-prebuilt.sh` so prebuilt installs get the same kiosk experience as source builds. The prebuilt installer is simpler (no TUI, no build step) so the integration is straightforward: add new functions, wire them into `main()`.

## Context

- `install-prebuilt.sh` is an independent installer for deploying prebuilt binaries from a payload directory
- Phase 28-31 created kiosk config reference files at `docs/pi-config/kiosk/`
- Phase 32-01 adds these same functions to `install.sh` (the source-build installer) in parallel -- no conflicts
- The prebuilt installer uses a simpler linear flow without TUI

## Tasks

### Task 1: Add create_kiosk_session() function
type="auto"

Add a `create_kiosk_session()` function to `install-prebuilt.sh` that:

1. Creates `/etc/openauto-kiosk/labwc/` directory
2. Writes kiosk `rc.xml` (from `docs/pi-config/kiosk/labwc/rc.xml` reference)
3. Writes kiosk `autostart` (from `docs/pi-config/kiosk/labwc/autostart` reference)
4. Writes kiosk `environment` (from `docs/pi-config/kiosk/labwc/environment` reference)
5. Creates `/usr/share/wayland-sessions/openauto-kiosk.desktop` XDG session file
6. Creates `/etc/lightdm/lightdm.conf.d/50-openauto-kiosk.conf` LightDM drop-in with `autologin-user=$USER` and `autologin-session=openauto-kiosk`
7. Writes AccountsService entry at `/var/lib/AccountsService/users/$USER`
8. Deploys `splash.png` to `/usr/share/openauto-prodigy/`

The function should be idempotent (safe to re-run). It should be conditional on `$AUTOSTART` being true (same prompt reuse as the existing "Start on boot?" prompt, but semantically upgraded to mean kiosk mode).

**Done when:** Function exists in install-prebuilt.sh, writes all required config files using heredocs/sudo tee pattern matching existing style.

### Task 2: Add configure_boot_splash() function
type="auto"

Add a `configure_boot_splash()` function that:

1. Checks if `rpi-splash-screen-support` is available via `apt-cache show`
2. Copies `splash.tga` from payload to `/lib/firmware/logo.tga`
3. Runs `configure-splash /lib/firmware/logo.tga`
4. Repairs cmdline.txt (re-adds `quiet`, `loglevel=0`, `logo.nologo`; removes `splash` Plymouth param)
5. Masks Plymouth services (plymouth-start, plymouth-quit, plymouth-quit-wait)
6. Runs `update-initramfs -u`
7. Verifies: hook exists, TGA in initramfs, cmdline.txt parameters correct

The function is non-fatal -- if any step fails, warn and continue. Boot splash is nice-to-have, not load-bearing.

Also add `rpi-splash-screen-support` and `swaybg` to the dependency list in `install_dependencies()`.

**Done when:** Function exists, follows the exact steps from `docs/pi-config/kiosk/boot-splash.md`, added to dependency list.

### Task 3: Wire new functions into main() and update service/prompts
type="auto"

1. Change the autostart prompt text from "Start OpenAuto Prodigy automatically on boot?" to "Enable kiosk mode? (dedicated boot experience) [Y/n]" to reflect the upgraded meaning
2. Update `create_service()` to remove wf-panel-pi kill/restore if present (the prebuilt installer's service template may not have it, but ensure it's simplified)
3. Add `create_kiosk_session` and `configure_boot_splash` calls to `main()` after `create_service`, conditional on `$AUTOSTART`
4. Add payload validation for `splash.png` and `splash.tga` in `require_payload()` (warn if missing, don't fail -- these are optional for non-kiosk installs)
5. Update `run_diagnostics()` to check kiosk session status if kiosk was enabled
6. Update the final output to show kiosk revert instructions when kiosk mode was enabled

**Done when:** Full install-prebuilt.sh flow includes kiosk setup when user opts in, diagnostics verify kiosk config, final summary includes revert instructions.

## Verification

- `bash -n install-prebuilt.sh` passes (no syntax errors)
- All kiosk config file heredocs match reference files in `docs/pi-config/kiosk/`
- New functions follow existing code style (info/ok/warn helpers, sudo tee pattern)
- Boot splash function follows exact steps from boot-splash.md
- Kiosk setup is conditional (only runs if user opts in)

## Success Criteria

- install-prebuilt.sh can set up a complete kiosk experience when user enables it
- Prebuilt and source installers produce identical kiosk configurations
- Boot splash is non-fatal (warns on failure, continues)
- All changes are idempotent (safe to re-run)
