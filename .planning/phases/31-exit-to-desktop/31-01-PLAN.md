  <name>Task 1: Add exitToDesktop() to ApplicationController</name>
  <files>src/ui/ApplicationController.hpp, src/ui/ApplicationController.cpp</files>
  <action>
Add an `exitToDesktop()` method to ApplicationController that uses dm-tool to switch to the standard desktop session.

**In ApplicationController.hpp:**
Add `Q_INVOKABLE void exitToDesktop();` to the public methods section, alongside the existing quit/restart/minimize methods.

**In ApplicationController.cpp:**
Add the includes for QProcess and QTimer (QTimer is already included via main.cpp patterns, but ApplicationController.cpp needs them too).

Implement exitToDesktop():
```cpp
void ApplicationController::exitToDesktop()
{
    qInfo() << "[App] Exit to desktop requested";

    // Tell LightDM to switch to the normal desktop session for current user
    QProcess::startDetached("dm-tool", {"switch-to-user",
        qEnvironmentVariable("USER", "matt"), "rpd-labwc"});

    // Delay quit to give dm-tool time to register the switch with LightDM.
    // If app quits immediately, labwc exits, and LightDM might restart the
    // kiosk session (autologin) before the switch request is processed.
    QTimer::singleShot(500, []() {
        QCoreApplication::quit();
    });
}
```

Key design decisions per ARCHITECTURE.md:
- `dm-tool switch-to-user` does an atomic session switch (Pattern 3)
- `qEnvironmentVariable("USER", "matt")` reads current user, falls back to matt
- `rpd-labwc` is the standard RPi desktop session name
- 500ms delay ensures LightDM registers the switch before kiosk session ends
- `QCoreApplication::quit()` triggers clean exit (exit code 0), so systemd does NOT restart (Restart=on-failure only triggers on non-zero exit)
- Uses QProcess::startDetached so dm-tool survives even if the app exits

**In GestureOverlay.qml:**
Add a "Desktop" ElevatedButton to the action buttons RowLayout, between the Day/Night button and the Close button:

```qml
ElevatedButton {
    objectName: "overlayDesktopBtn"
    text: "Desktop"
    iconCode: "\uef6e"  // desktop_windows
    buttonEnabled: overlay.acceptInput
    implicitWidth: UiMetrics.overlayBtnW
    implicitHeight: UiMetrics.overlayBtnH
    onClicked: {
        ActionRegistry.dispatch("app.exitToDesktop")
        overlay.dismiss()
    }
}
```

The button order becomes: Home, Day/Night, Desktop, Close.
- `\uef6e` is the Material Icons codepoint for desktop_windows
- The button dispatches the action then dismisses the overlay
- Uses same sizing as other overlay buttons (UiMetrics.overlayBtnW/H)
