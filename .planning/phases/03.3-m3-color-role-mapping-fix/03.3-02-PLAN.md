  <action>
    Apply the M3 surface container hierarchy. Each element gets a container level based on its visual elevation in the UI stack:

    **surfaceContainer (mid-level -- navigation bars, standard containers):**
    - `Navbar.qml`: Change navbar background from `surface` (or `outline` if that's current) to `surfaceContainer`

    **surfaceContainerHigh (elevated -- dialogs, panels):**
    - `ExitDialog.qml`: Change background from `surfaceVariant` to `surfaceContainerHigh`
    - `PairingDialog.qml`: Change background to `surfaceContainerHigh`
    - `NotificationArea.qml`: Change panel background from `surfaceContainerLow @87%` to `surfaceContainerHigh @87%` (keep alpha). Use `Qt.rgba(ThemeService.surfaceContainerHigh.r, ThemeService.surfaceContainerHigh.g, ThemeService.surfaceContainerHigh.b, 0.87)`
    - `NavbarPopup.qml`: Power menu background from `surface` to `surfaceContainerHigh`. Border from `barShadow` to `outlineVariant`.

    **surfaceContainerHighest (top-level overlays):**
    - `GestureOverlay.qml`: Change background from `surfaceContainerLow @87%` to `surfaceContainerHighest @87%` (keep alpha)
    - `AndroidAutoMenu.qml`: If it has an overlay background, use `surfaceContainerHighest`

    **surfaceContainerLow (cards, list items, button resting states -- stays for most):**
    - Buttons at resting state: KEEP `surfaceContainerLow` (this is correct per M3)
    - Cards/list items: Where `surfaceVariant` is used for card/panel backgrounds, change to `surfaceContainerLow`
    - `FullScreenPicker.qml`: Item backgrounds from `surfaceVariant` to `surfaceContainerLow`
    - `EqBandSlider.qml`: Visualizer/panel background from `surfaceVariant` to `surfaceContainerLow`
    - `PhoneView.qml`: Dial pad backgrounds from `surfaceVariant` to `surfaceContainerLow`
    - Settings pages: Any `surfaceVariant` panel backgrounds to `surfaceContainerLow`

    **surfaceVariant disposition:**
    - Most `surfaceVariant` refs become `surfaceContainerLow` (card/panel backgrounds)
    - Exception: `NavbarPopup.qml` content area can keep `surfaceVariant` if it's semantically a variant surface within a container
    - After remapping, run: `grep -rn "ThemeService\.surfaceVariant" qml/` to verify remaining uses are intentional

    **Do NOT touch:** Button pressed states or toggle states (that's Task 2). Only remap static/resting backgrounds in this task.
  </action>

    **Button pressed state pattern change (~20 occurrences):**
    Find all instances of the pattern:
    ```qml
    color: parent.pressed ? ThemeService.primary : ThemeService.surfaceContainerLow
    ```
    Replace with:
    ```qml
    color: parent.pressed ? ThemeService.primaryContainer : ThemeService.surfaceContainerLow
    ```

    Also find text color bindings inside pressed buttons. Where text uses `ThemeService.background` or implicit color on pressed buttons, add explicit pressed text color:
    ```qml
    color: parent.pressed ? ThemeService.onPrimaryContainer : ThemeService.onSurface
    ```

    **Error/destructive button pressed states:**
    For buttons that use `error` color (delete, disconnect, reject), the pressed state pattern is:
    ```qml
    color: parent.pressed ? Qt.darker(ThemeService.error, 1.15) : ThemeService.error
    ```
    Text on these stays `ThemeService.onError`. Do NOT change error buttons to use primaryContainer -- they should stay error-colored.

    **Toggle/Switch selected states:**
    - `SettingsToggle.qml`: If the Switch or toggle uses `ThemeService.primary` for the active/selected state, change to `ThemeService.secondaryContainer`. Note: Qt Quick Controls Switch styling may use `Material.accent` set in main.qml -- if so, the toggle styling may need to be handled via explicit palette overrides rather than just color bindings. Check the actual implementation.

    **Segmented button selected states:**
    - `SegmentedButton.qml`: Selected segment background from `primary` to `secondaryContainer`. Selected segment text from `background`/`onPrimary` to `onSecondaryContainer`. Unselected segment background from `surfaceVariant` to `surfaceContainerLow`.

    **Remove pressed derived color from ThemeService:**
    - `ThemeService.hpp`: The `pressed` Q_PROPERTY and accessor should already be removed by Plan 01 Task 1. Verify it's gone.
    - `ThemeService.cpp`: Remove `pressed` computation from `recalcDerivedColors()` if still present. Keep `barShadow`, `success`, `onSuccess` computations.
