  <action>
    Apply the press feedback pattern to all tappable controls. The pattern per user decisions is scale+opacity driven by MouseArea `pressed` state, with `Behavior` animations at `UiMetrics.animDurationFast` (80ms) and `Easing.OutCubic`.

    **SettingsListItem.qml:**
    - The root Item needs to animate. Add `scale` and `opacity` properties bound to the existing MouseArea's `pressed` state:
      ```qml
      scale: mouseArea.pressed ? 0.97 : 1.0
      opacity: mouseArea.pressed ? 0.85 : 1.0
      Behavior on scale { NumberAnimation { duration: UiMetrics.animDurationFast; easing.type: Easing.OutCubic } }
      Behavior on opacity { NumberAnimation { duration: UiMetrics.animDurationFast; easing.type: Easing.OutCubic } }
      ```
    - Give the MouseArea an id (e.g., `mouseArea`) if it doesn't have one.
    - Replace the divider's `color: ThemeService.descriptionFontColor` + `opacity: 0.15` with `color: ThemeService.dividerColor` + `opacity: 1.0`.

    **Tile.qml:**
    - Replace the hover highlight approach (`containsMouse ? highlightColor : controlBackgroundColor`) with press feedback:
      ```qml
      scale: mouseArea.pressed ? 0.95 : 1.0
      opacity: mouseArea.pressed && tileEnabled ? 0.85 : (tileEnabled ? 1.0 : 0.5)
      ```
    - Add Behavior on scale and opacity (same pattern).
    - Keep the existing `mouseArea` id. Remove `hoverEnabled` and `containsMouse` usage (not useful on touchscreen).
    - Use 0.95 for tiles (slightly more pronounced than list items since tiles are larger).

    **FullScreenPicker.qml:**
    - Add press feedback to the picker row delegate items inside the Dialog ListView. Each delegate Item needs scale+opacity bound to its own MouseArea pressed state.
    - Also add press feedback to the main tappable row (the one that opens the picker) — it already has a MouseArea.

    **SegmentedButton.qml:**
    - Add press feedback to each segment Rectangle. The MouseArea is already inside each Repeater delegate.
    - Use subtle feedback (0.97 scale) since segments are small.
    - Give each segment's MouseArea an id.

    **For all controls:** Use `NumberAnimation` (not `ScaleAnimator`/`OpacityAnimator`) inside `Behavior` — per research, Behavior blocks work with regular animations and 80ms is too short for render-thread offloading to matter.

    **Do NOT** apply press feedback to SettingsSlider (the thumb drag IS the feedback — per research pitfall 3) or SettingsToggle (the Switch toggle IS the feedback).
