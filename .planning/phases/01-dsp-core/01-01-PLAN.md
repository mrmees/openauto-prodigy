  <name>Task 1: BiquadFilter — RBJ peaking EQ with DF2T processing</name>
  <files>src/core/audio/BiquadFilter.hpp, tests/test_biquad_filter.cpp, tests/CMakeLists.txt</files>
  <behavior>
    - calcPeakingEQ(1000Hz, +6dB, Q=1.4142, 48000Hz) produces coefficients where b0 != 1.0, a1 != 0, a2 != 0 (non-trivial filter)
    - calcPeakingEQ(1000Hz, 0dB, Q=1.4142, 48000Hz) produces unity coefficients (b0≈1, b1≈b1_pass, b2≈b2_pass, a1≈a1_pass, a2≈a2_pass — effectively passthrough)
    - Processing a 1kHz sine wave through a +12dB peaking filter at 1kHz produces output with higher amplitude than input (within 1dB of expected +12dB gain)
    - Processing a 1kHz sine wave through a +12dB peaking filter at 100Hz produces output with approximately unity amplitude (filter doesn't affect frequencies far from center)
    - Processing a 100Hz sine wave through a +12dB peaking filter at 100Hz produces boosted output
    - Left and right channels processed through same BiquadFilter coefficients but independent state produce identical output for identical input (no crosstalk between stereo state pairs)
    - Filter state (z1, z2) is properly maintained across multiple processSample calls (continuous streaming, not just single samples)
    - resetState() zeros z1 and z2
  </behavior>
  <action>
    **Header (`src/core/audio/BiquadFilter.hpp`):**

    Create a header-only implementation in `namespace oap` containing:

    1. **`struct BiquadCoeffs`** — 5 normalized coefficients (b0, b1, b2, a1, a2) as `float`. Add a static `unity()` factory returning passthrough coefficients (b0=1, others=0).

    2. **`BiquadCoeffs calcPeakingEQ(float freqHz, float gainDb, float Q, float sampleRate)`** — free function implementing the RBJ Audio EQ Cookbook peaking EQ formula:
       - `A = pow(10, gainDb / 40)`
       - `w0 = 2 * PI * freqHz / sampleRate`
       - `alpha = sin(w0) / (2 * Q)`
       - Compute b0, b1, b2, a0, a1, a2 per cookbook
       - Return coefficients normalized by a0 (divide all by a0)

    3. **`struct BiquadState`** — two float state variables `z1`, `z2` initialized to 0. Method `reset()` zeros both.

    4. **`inline float processSample(float x, const BiquadCoeffs& c, BiquadState& s)`** — Direct Form II Transposed:
       - `y = c.b0 * x + s.z1`
       - `s.z1 = c.b1 * x - c.a1 * y + s.z2`
       - `s.z2 = c.b2 * x - c.a2 * y`
       - return y

    5. **Constants:** `kGraphicEqQ = 1.4142f`, `kCenterFreqs[10] = {31, 62, 125, 250, 500, 1000, 2000, 4000, 8000, 16000}`, `kNumBands = 10`

    Use `#include <cmath>` for math functions. Use `#ifndef M_PI` guard for M_PI definition.

    **Test (`tests/test_biquad_filter.cpp`):**

    Qt Test class `TestBiquadFilter` with tests for each behavior above. Use steady-state sine wave measurement: generate N samples of `sin(2*PI*freq*n/sampleRate)`, process through filter, measure RMS of last 1024 samples (skip transient), compare to input RMS. Convert ratio to dB: `20*log10(outRms/inRms)`.

    For the stereo independence test: process same input through two independent BiquadState instances with same coefficients, verify identical output.

    **CMake:** Add to `tests/CMakeLists.txt` under a new `# --- DSP tests ---` section:
    ```cmake

    ```

    **Per user decisions:**

    - DF2T topology as recommended by research

<task type="auto" tdd="true">
  <name>Task 2: SoftLimiter — envelope-follower peak limiter</name>
  <files>src/core/audio/SoftLimiter.hpp, tests/test_soft_limiter.cpp, tests/CMakeLists.txt</files>
  <behavior>
    - Signal at -6dBFS (amplitude 0.5) passes through with unity gain (output ≈ input within 0.01)
    - Signal at +6dBFS (amplitude 2.0) is reduced to approximately 0dBFS (output peak ≤ 1.05, allowing small overshoot from envelope follower)
    - After a loud transient, limiter releases back to unity gain within ~100ms (release time ~50ms, full recovery slightly longer)
    - Attack is fast: a sudden +12dB spike is caught within ~1ms (24 samples at 48kHz)
    - reset() returns limiter to initial state (envelope = 0)
    - Limiter preserves signal polarity (negative input produces negative output)
    - Limiter is per-channel: two instances can process L/R independently
  </behavior>
  <action>
    **Header (`src/core/audio/SoftLimiter.hpp`):**

    Create a header-only implementation in `namespace oap`:

    **`class SoftLimiter`** with:
    - Constructor taking `float sampleRate = 48000.0f` — computes attack and release coefficients:
      - Attack: ~0.5ms → `attackCoeff_ = 1.0f - std::exp(-1.0f / (0.0005f * sampleRate))`
      - Release: ~50ms → `releaseCoeff_ = 1.0f - std::exp(-1.0f / (0.05f * sampleRate))`
      - Threshold: `1.0f` (0dBFS)
    - `float process(float x)`:
      - Track peak envelope: `absX = std::fabs(x)`
      - If `absX > envelope_`: `envelope_ += attackCoeff_ * (absX - envelope_)` (fast attack)
      - Else: `envelope_ += releaseCoeff_ * (absX - envelope_)` (slow release)
      - If `envelope_ > threshold_`: `return x * (threshold_ / envelope_)` (gain reduction)
      - Else: `return x`
    - `void reset()` — set `envelope_ = 0.0f`
    - Private members: `envelope_` (float, init 0), `attackCoeff_`, `releaseCoeff_`, `threshold_`

    **Test (`tests/test_soft_limiter.cpp`):**

    Qt Test class `TestSoftLimiter` with tests for each behavior above.

    Test approach:
    - Generate constant-amplitude sine waves at various levels
    - Process through limiter
    - Measure output peak/RMS after steady state (skip first 4800 samples / 100ms for envelope to settle)
    - For attack test: step from silence to loud signal, measure how many samples until output is limited
    - For release test: step from loud to quiet, measure how many samples until gain returns to unity

    **CMake:** Add to `tests/CMakeLists.txt` DSP section:
    ```cmake
    oap_add_test(test_soft_limiter SOURCES test_soft_limiter.cpp)
    ```

    **Per user decisions:**
    - Soft limiter prevents digital crackle — auto-attenuates transparently
    - Per-channel (not stereo-linked) per research recommendation — simpler, no crosstalk
  </action>
