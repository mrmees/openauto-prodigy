  <name>Task 1: Rewrite HomeMenu.qml with SwipeView multi-page grid, PageIndicator, and page management</name>

  <action>
    Rewrite HomeMenu.qml to wrap the grid content inside a SwipeView with per-page Repeaters, add PageIndicator between grid and dock, and add page management FABs. The existing edit mode, drag/resize, picker overlay, and toast functionality MUST be preserved exactly.

    **Structure (ColumnLayout children in order):**
    1. `SwipeView` (id: pageView) — fills available space
    2. `PageIndicator` — centered horizontally, between SwipeView and dock
    3. `LauncherDock` — fixed at bottom (unchanged)

    **SwipeView configuration:**
    - `interactive: !homeScreen.editMode` (PAGE-04: disable swipe during edit)
    - `clip: true`
    - `Component.onCompleted`: set `contentItem.boundsBehavior = Flickable.DragAndOvershootBounds` for rubber band overscroll, set `contentItem.highlightMoveDuration = 350` for ~350ms glide animation
    - Bind `onCurrentIndexChanged` to update `WidgetGridModel.activePage = currentIndex`

    **Page content (inside SwipeView):**
    - Use `Repeater { model: WidgetGridModel.pageCount }` to generate page items
    - Each page item contains a `Loader` with lazy activation: `active: SwipeView.isCurrentItem || SwipeView.isNextItem || SwipeView.isPreviousItem` (PAGE-08)
    - Use inline `Component` (NOT separate file) to avoid SKIP_CACHEGEN issues per project gotcha
    - The loaded component is the grid content — the `gridContainer` Item with cellWidth/cellHeight, background MouseArea, dotted grid overlay, and widget Repeater
    - Widget Repeater delegate filters by page: `visible: model.page === pageIndex && model.visible` (use delegateItem's visibility, not Item visibility, to avoid scene graph overhead issues)
    - Pass `pageIndex` from Loader to grid content via a property: `property int pageIndex: index`

    **CRITICAL: Drag overlay, drop highlight, drag placeholder, resize ghost, FABs, picker overlay, and toast MUST remain at the homeScreen level (outside SwipeView).** These overlay the entire screen, not just one page. The drag overlay at z:200 needs to capture touches across the full area. If placed inside a SwipeView page, clipping will break drag behavior. Reference gridContainer properties from the current page's loaded content.

    **Handling gridContainer reference for overlays:**
    Since gridContainer is now inside a Loader inside SwipeView, the drag overlay needs access to cellWidth/cellHeight. Approach: define `cellWidth` and `cellHeight` as properties on homeScreen computed from SwipeView dimensions and grid model: `readonly property real cellWidth: pageView.width / WidgetGridModel.gridColumns` and `readonly property real cellHeight: (pageView.height) / WidgetGridModel.gridRows`. The drag overlay uses these homeScreen-level properties. The per-page grid delegates also use these same values for positioning.

    **PageIndicator:**
    - `count: pageView.count`
    - `currentIndex: pageView.currentIndex`
    - `interactive: true` — always tappable, including during edit mode (PAGE-07)
    - Custom delegate: filled circle, `implicitWidth: UiMetrics.spacing * 1.2`, `implicitHeight: implicitWidth`, `radius: width / 2`
    - Active dot: `color: ThemeService.primary, opacity: 1.0`
    - Inactive dot: `color: ThemeService.onSurfaceVariant, opacity: 0.4`
    - `Behavior on opacity { OpacityAnimator { duration: 150 } }` for smooth transition
    - `onCurrentIndexChanged: if (currentIndex !== pageView.currentIndex) pageView.setCurrentIndex(currentIndex)` — use setCurrentIndex method to avoid QTBUG-80750

    **Add-page FAB (edit mode only):**
    - Position: above the existing add-widget FAB (stack vertically on the right side)
    - Same styling as add-widget FAB but with a page+ icon (use "\ue89c" note_add or "\ue85d" library_add — choose whichever reads as "add page")
    - `visible: homeScreen.editMode`
    - `onClicked`: call `WidgetGridModel.addPage()`, then `pageView.setCurrentIndex(WidgetGridModel.pageCount - 1)` to navigate to new page, restart inactivity timer

    **Delete-page FAB (edit mode only, when pageCount > 1 and not on page 0):**
    - Position: left side of screen, vertically centered or near bottom
    - Distinct styling: use error color background with delete icon ("\ue872" delete)
    - `visible: homeScreen.editMode && WidgetGridModel.pageCount > 1 && pageView.currentIndex > 0`
    - `onClicked`: show confirmation dialog (don't delete immediately)

    **Confirmation dialog (deletePageDialog):**
    - Follows existing overlay pattern (scrim + centered card)
    - `z: 300` (above picker overlay)
    - Shows: "Delete page and N widgets?" where N = `WidgetGridModel.widgetCountOnPage(targetPage)`
    - Two buttons: "Cancel" (closes dialog) and "Delete" (error-colored, performs deletion)
    - Delete action: save currentIndex, call `WidgetGridModel.removePage(targetPage)`, navigate to min(savedIndex, pageCount-1), close dialog, restart timer
    - Use NormalText for message, UiMetrics for sizing, ThemeService for colors

    **Empty page auto-cleanup on edit mode exit:**
    In `exitEditMode()`, after existing cleanup, use `Qt.callLater()` to defer page cleanup (avoid visual glitch during SwipeView animation per research pitfall):
    ```
    Qt.callLater(function() {
        // Clean from highest to lowest to avoid index shifting issues
        for (var i = WidgetGridModel.pageCount - 1; i > 0; i--) {
            if (WidgetGridModel.widgetCountOnPage(i) === 0) {
                WidgetGridModel.removePage(i)
            }
        }
        // Ensure current page is valid
        if (pageView.currentIndex >= WidgetGridModel.pageCount)
            pageView.setCurrentIndex(WidgetGridModel.pageCount - 1)
    })
    ```
    Note: page 0 is never cleaned (cannot be deleted).

    **Edit mode activePage sync:**
    When SwipeView page changes, update `WidgetGridModel.activePage`. This ensures placeWidget/canPlace/findFirstAvailableCell operate on the visible page. Add a binding or connection: `pageView.onCurrentIndexChanged: WidgetGridModel.activePage = pageView.currentIndex`

    - [ ] AA fullscreen auto-exit (PluginModel connection)

    - [ ] Dotted grid lines in edit mode
