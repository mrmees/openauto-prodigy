Replace the `create_system_service()` function (currently lines ~1656-1711) with a new implementation that:

**1. Group management (before anything else):**
```bash
# Create openauto group if it doesn't exist
if ! getent group openauto >/dev/null 2>&1; then
    sudo groupadd openauto
    ok "Created openauto group"
fi

# Add installing user to openauto group
if ! id -nG "$USER" | grep -qw openauto; then
    sudo usermod -aG openauto "$USER"
    ok "Added $USER to openauto group"
    local GROUP_CHANGED=true
fi
```

**2. Upgrade detection and migration:**
```bash
local UNIT_PATH="/etc/systemd/system/openauto-system.service"
local MIGRATING=false

if [[ -f "$UNIT_PATH" ]]; then
    # Detect old unprivileged unit (has User= line that is NOT root)
    # NOTE: Use sed, not grep -oP -- Perl regex is not available on minimal Trixie
    local OLD_USER
    OLD_USER=$(sed -n 's/^User=//p' "$UNIT_PATH" 2>/dev/null || echo "")
    if [[ -n "$OLD_USER" && "$OLD_USER" != "root" ]]; then
        MIGRATING=true
        info "Migrating system service from User=$OLD_USER to User=root"

        # Step 1: Stop old service cleanly
        sudo systemctl stop openauto-system 2>/dev/null || true

        # Step 2: Clean stale iptables proxy state (explicit, not dependent on Phase 16 self-heal)
        sudo iptables -t nat -D OUTPUT -p tcp -j OPENAUTO_PROXY 2>/dev/null || true
        sudo iptables -t nat -F OPENAUTO_PROXY 2>/dev/null || true
        sudo iptables -t nat -X OPENAUTO_PROXY 2>/dev/null || true
        info "Cleaned stale iptables proxy state"
    fi
fi
```

**3. Python venv setup** (keep existing logic -- try venv, fall back to system python).

**4. Template rendering:**
Replace the inline heredoc unit file with sed-based template rendering:
```bash
local TEMPLATE="$SYS_DIR/../system-service/openauto-system.service.in"
if [[ ! -f "$TEMPLATE" ]]; then
    TEMPLATE="$SYS_DIR/openauto-system.service.in"
fi
# Fall back: if running from repo, use repo-relative path
if [[ ! -f "$TEMPLATE" ]]; then
    TEMPLATE="$(dirname "$0")/system-service/openauto-system.service.in"
fi

if [[ ! -f "$TEMPLATE" ]]; then
    warn "Service template not found -- using inline fallback"
    # Keep existing inline heredoc as fallback, but with User=root and hardening
else
    sed -e "s|@@PYTHON_PATH@@|$PYTHON_PATH|g" \
        -e "s|@@SYS_DIR@@|$SYS_DIR|g" \
        "$TEMPLATE" | sudo tee "$UNIT_PATH" > /dev/null
fi
```

**5. Reload and enable:**
```bash
sudo systemctl daemon-reload
sudo systemctl enable --quiet openauto-system
```

**6. Post-migration notice:**
```bash
if [[ "$MIGRATING" == "true" ]]; then
    ok "System service migrated: now runs as root with restricted IPC socket"
fi

if [[ "${GROUP_CHANGED:-false}" == "true" ]]; then
    warn "NOTE: You were added to the 'openauto' group."
    warn "Group membership takes effect after logging out and back in (or rebooting)."
    warn "Until then, the Qt client may not be able to connect to the system service socket."
fi

ok "System service installed and enabled"
```

**CRITICAL: Do NOT create, mkdir, chown, or chmod /run/openauto in the installer.** The systemd unit has `RuntimeDirectory=openauto` which creates the directory on service start with correct ownership. The daemon's `ipc_server.py` enforces socket mode/ownership after bind (0660, root:openauto). Installer-side /run/openauto prep is redundant on tmpfs and creates confusing split responsibility. The installer's job is: group creation, user membership, unit file rendering, daemon-reload. The daemon's job is: runtime directory contents (socket mode/ownership).

**Key constraints (from user decisions):**
- Migration is auto with notice, NO interactive prompt
- Stale iptables cleanup is explicit (does not rely on Phase 16 startup self-heal)
- Session refresh warning must be honest -- do not claim socket access works immediately
- One source of truth: template file, not inline heredoc (except fallback)
- IMPORTANT: Do NOT use `grep -oP` (Perl regex) -- not available on minimal Trixie. Use `sed -n 's/^User=//p'` for portable extraction.

**Post-install verification scope:** After `usermod -aG openauto $USER`, the Qt client CANNOT connect to the new socket in the SAME login session. Group membership requires a session refresh (logout/login or reboot). The installer prints this warning honestly. Verifying actual client connectivity post-install is OUT OF SCOPE for this phase unless the session is refreshed first.
