<objective>
Establish the daemon privilege model and IPC socket lockdown.

Purpose: The system daemon needs root to perform iptables, systemctl, rfkill, and SDP permission operations. The IPC socket must be restricted so only authorized clients (root or openauto group members) can invoke privileged methods. This is the security foundation for all subsequent proxy routing work.

Output: Service template file (`openauto-system.service.in`), hardened IPC server with injectable peer credential validation, and tests proving the authorization policy works deterministically via mocked caller identities.
</objective>

**IPC server peer credential auth (`ipc_server.py`):**

1. Add `socket`, `struct`, `grp`, `pwd` imports at top.

2. Add a module-level function `check_peer_authorized(sock) -> tuple[bool, dict]`:
   - Use `sock.getsockopt(socket.SOL_SOCKET, socket.SO_PEERCRED, struct.calcsize('iII'))` to get peer pid, uid, gid.
   - Parse with `struct.unpack('iII', cred_data)` -> (pid, uid, gid).
   - Authorization policy (per user decision -- NON-NEGOTIABLE):
     - uid == 0 (root): authorized
     - Otherwise: resolve `openauto` group GID via `grp.getgrnam('openauto').gr_gid`. Check if peer uid's user is a member of the openauto group using `grp.getgrnam('openauto').gr_mem` (check if username from `pwd.getpwuid(uid).pw_name` is in members list) OR peer's gid matches the openauto group gid.
     - If `openauto` group doesn't exist (KeyError from getgrnam): unauthorized unless root.
   - Return `(authorized: bool, cred_info: dict)` where cred_info has pid, uid, gid keys.
   - **This function is module-level (not a method) so tests can mock it directly.**

3. Add an `__init__` parameter `authorizer: Callable | None = None`. If provided, store as `self._authorizer`. If None, default to `check_peer_authorized`. This is the dependency injection seam for testing.

4. Modify `_handle_client` to check peer credentials immediately on connection:
   - Get the underlying socket from the writer's transport: `sock = writer.get_extra_info('socket')`
   - Call `self._authorizer(sock)` to get `(authorized, cred_info)`.
   - If unauthorized: log at warning level with pid/uid/gid, send `{"error": {"code": -32600, "message": "permission denied"}}` + newline, close connection, return.
   - If authorized: proceed with existing message loop.

5. Change `start()` method:
   - After `asyncio.start_unix_server(...)`, change `os.chmod(self._socket_path, 0o666)` to `os.chmod(self._socket_path, 0o660)`.
   - After chmod, call `os.chown(self._socket_path, 0, grp.getgrnam('openauto').gr_gid)` to set root:openauto ownership.
   - Wrap the chown in try/except KeyError (group doesn't exist yet) -- log a warning but don't crash. In that case the socket keeps its default ownership (the daemon's uid:gid, which is root when running as root).
   - Also wrap in try/except PermissionError for environments where the daemon isn't root (test environments) -- log debug, don't crash.

**IMPORTANT:** Do NOT change the `_dispatch` method, `register_method`, or any other existing API. The Qt client (`SystemServiceClient`) does not need changes -- it connects via `QLocalSocket` and will work as long as the user running the Qt app is in the `openauto` group.

**IMPORTANT:** Do NOT add an `auth_enabled` boolean toggle. Use the `authorizer` callable injection instead -- tests inject a mock/stub authorizer, production uses the default `check_peer_authorized`. This avoids having a "disable security" flag in production code.
