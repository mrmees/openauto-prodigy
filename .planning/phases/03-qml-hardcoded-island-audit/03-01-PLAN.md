# Plan 03-01: New UiMetrics Tokens + Shell Chrome Tokenization

## Objective

Add ~8 new UiMetrics tokens to cover all remaining hardcoded dimension categories, register them in YamlConfig, then tokenize the shell chrome layer (TopBar, BottomBar, NavStrip, NotificationArea). This plan must complete before Plans 02 and 03, which consume the new tokens.

<context>
## Phase Goal
Every QML file uses UiMetrics tokens for all sizing -- zero hardcoded pixel values remain (excluding debug overlays).

## Key Decisions (from CONTEXT.md)
- Minimal new tokens (~6-8). Reuse existing tokens where close enough.
- New tokens follow Phase 1 pattern: overridable via `ui.tokens.*`, registered in YamlConfig initDefaults().
- All new tokens added upfront in Plan 1 before any file edits.
- Proportional scaling with safety floors: call/overlay buttons have touchMin floors.
- Divider height: keep at literal 1 (standard practice, scaled value rounds to 0 at small res).
- Radius strategy: 3 tiers -- radiusSmall (8), radius (10, existing), radiusLarge (16).
- Spacing: reuse existing tokens (spacing=8, marginRow=12, gap=16, marginPage=20, sectionGap=20). No new spacing tokens.

## Discretion Decisions Made
- **Button tokens:** `callBtnSize` (72 base) with `Math.max(computed, touchMin)` floor. One token for both PhoneView and IncomingCallOverlay.
- **Overlay buttons:** Separate `overlayBtnW` (100) and `overlayBtnH` (36, touchMin floor) for GestureOverlay.
- **statusDot** (10) and **progressH** (4): non-overridable Pattern C tokens (small internal dims).
- **albumArt** (200): overridable Pattern A token.

## Anti-Patterns (MUST AVOID)
- Do NOT modify existing token names or base values (Pitfall 1).
- Do NOT forget `Math.round()` on any inline scaled expression (Pitfall 2).
- Do NOT use font scale for layout tokens or layout scale for fonts (Pitfall 3).
- Do NOT tokenize `height: 1` dividers -- leave as literal 1 (research decision).
- Do NOT change `main.qml` width/height (Pitfall 6 -- circular dependency).
- Do NOT modify files that already correctly use UiMetrics tokens unless they ALSO have hardcoded values.
