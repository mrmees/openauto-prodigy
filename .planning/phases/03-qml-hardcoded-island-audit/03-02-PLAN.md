# Plan 03-02: Plugin View Tokenization

## Objective

Replace all hardcoded pixel values in the 5 plugin view files (Sidebar, GestureOverlay, PhoneView, IncomingCallOverlay, BtAudioView) with UiMetrics tokens. These are the worst offenders with 20+ hardcoded values each.

<context>
## Phase Goal
Every QML file uses UiMetrics tokens for all sizing -- zero hardcoded pixel values remain.

## Key Decisions
- Proportional scaling with safety floors: call/overlay buttons have touchMin floors.
- Album art uses `UiMetrics.albumArt` (base 200, overridable).
- Call buttons use `UiMetrics.callBtnSize` (base 72, touchMin floor).
- GestureOverlay buttons use `UiMetrics.overlayBtnW` / `overlayBtnH`.
- Status indicators use `UiMetrics.statusDot` (base 10, non-overridable).
- Progress bar uses `UiMetrics.progressH` (base 4, non-overridable).
- Radius: 3 tiers -- `radiusSmall` (8), `radius` (10), `radiusLarge` (16).
- Spacing: reuse existing tokens only.
- Icons: iconSmall (20), iconSize (36), inline `Math.round(N * UiMetrics.scale)` for other sizes.

## New tokens from Plan 01 (available)
- `UiMetrics.radiusSmall` (base 8)
- `UiMetrics.radiusLarge` (base 16)
- `UiMetrics.albumArt` (base 200)
- `UiMetrics.callBtnSize` (base 72, floor = touchMin)
- `UiMetrics.overlayBtnW` (base 100)
- `UiMetrics.overlayBtnH` (base 36, floor = touchMin)
- `UiMetrics.statusDot` (base 10)
- `UiMetrics.progressH` (base 4)

## Existing tokens (for reference)
- Spacing: `spacing` (8), `marginRow` (12), `gap` (16), `marginPage` (20), `sectionGap` (20)
- Fonts: `fontTiny` (14), `fontSmall` (16), `fontBody` (20), `fontTitle` (22), `fontHeading` (28)
- Icons: `iconSmall` (20), `iconSize` (36)
- Layout: `touchMin` (56), `trackThick` (6), `knobSize` (22), `knobSizeSmall` (18)
- Radii: `radius` (10), `radiusSmall` (8), `radiusLarge` (16)
</context>

## Anti-Patterns (MUST AVOID)
- Do NOT forget `Math.round()` on inline scaled expressions.
- Do NOT use `_fontScaleTotal` or font tokens for layout dimensions -- use `UiMetrics.scale`.
- Do NOT modify debug overlay in AndroidAutoMenu.qml (Pitfall 4).
- Do NOT replace `height: 1` dividers, `padding: 0`, `border.width: 1/2`.
- Do NOT change existing UiMetrics token references that are already correct.
