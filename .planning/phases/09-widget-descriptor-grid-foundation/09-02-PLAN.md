<objective>
Replace the fixed-pixel grid math in DisplayInfo with diagonal-proportional DPI-based cell sizing, move grid frame computation (cols/rows/offsets) to QML, remove the hardcoded 56px dock height deduction, and add gridDensityBias config support.

Purpose: GL-01 requires physically consistent cell sizing across displays. GL-02 requires removing the dock deduction. Together they make the grid layout DPI-aware and ready for the dock-free future (Phase 10).
Output: Rewritten DisplayInfo (cellSide only), updated HomeMenu.qml (grid frame owner + dock z-order), updated main.cpp (QScreen wiring), updated tests.

**Note on physical sizing math (GL-01):** The diagonal-proportional formula `cellSide = diagPx / divisor` inherently achieves physical consistency because `physical_mm = diagPx / DPI = (cellSide * divisor) / DPI`. Since the divisor is a fixed proportion of the diagonal, and `diagPx / DPI = physical_diagonal_mm`, the cell's physical size is always `physical_diagonal_mm / divisor` -- resolution cancels out completely. All three DPI cascade paths (config override, QScreen, pixel heuristic) produce the same formula because the approach is resolution-independent by design. The cascade is structural scaffolding for a future mm-based targeting path where DPI would produce different output; today it validates trustworthiness without changing the result.
</objective>
