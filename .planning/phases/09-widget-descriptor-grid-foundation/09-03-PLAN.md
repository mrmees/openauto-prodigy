  <behavior>
    YAML persistence:
    - `gridSavedCols()` returns `widget_grid.grid_cols` (default 0 = never saved)
    - `gridSavedRows()` returns `widget_grid.grid_rows` (default 0 = never saved)
    - `setGridSavedDims(cols, rows)` writes both fields

    Dim persistence policy (per CONTEXT.md: "persists only when user explicitly edits"):
    - `setGridPlacements()` does NOT auto-write grid dims
    - When the user explicitly edits (moves/resizes/adds/removes a widget), the save path writes BOTH placements AND current grid dims together by calling `setGridPlacements()` then `setGridSavedDims()`
    - This means dims are written at the same time as placements, triggered by user edit actions

    Base snapshot architecture:
    - WidgetGridModel stores `basePlacements_` (from YAML) and `livePlacements_` (displayed)
    - `placements_` becomes `livePlacements_` (rename for clarity)
    - New `basePlacements_` member: loaded from YAML, only updated on explicit save
    - New `savedCols_`, `savedRows_` members: grid dims when base was last saved
    - `setPlacements()` sets both base and live (initial load)
    - `setGridDimensions()`: if new dims differ from savedCols_/savedRows_, call remapPlacements() from base -> live; if same, live = base copy

    Remap algorithm (private method `remapPlacements`):
    - Input: basePlacements_, savedCols_, savedRows_, newCols, newRows, registry_
    - For each base placement, in order (first-placed wins):
      1. Proportional position: newCol = qRound(baseCol * newCols / savedCols), newRow = qRound(baseRow * newRows / savedRows)
      2. Spans stay original size (do NOT scale)
      3. Clamp: col = min(col, max(0, newCols - colSpan)), row = min(row, max(0, newRows - rowSpan))
      4. Check min span vs grid: if descriptor.minCols > newCols or descriptor.minRows > newRows, mark visible=false, add warning, continue
      5. Check overlap with already-placed widgets on same page using occupancy grid
      6. If overlap, try spiral nudge: scan positions outward from target (right, down-right, down, down-left, left, up-left, up, up-right), increasing radius. First non-overlapping position within bounds wins.
      7. If no position found on page, spill to next page at position (0,0). Auto-create page if needed.
      8. Mark placed position in occupancy

    Edit mode deferral:
    - Add `bool editMode_` flag to WidgetGridModel
    - Add `Q_INVOKABLE void setEditMode(bool editing)`
    - When `setGridDimensions()` is called during edit mode, queue the dims but don't remap
    - When edit mode exits (setEditMode(false)), apply queued remap if pending

    Save behavior:
    - When user edits (placeWidget, moveWidget, resizeWidget, removeWidget), the live state becomes the new base: `basePlacements_ = livePlacements_`, `savedCols_ = cols_`, `savedRows_ = rows_`
    - The save-to-YAML path (already exists, called from QML) writes placements AND grid dims: calls `yamlConfig->setGridPlacements(...)` then `yamlConfig->setGridSavedDims(cols_, rows_)`

    Test expectations (test_widget_grid_remap.cpp):
    - Same dims: live = base (no remap)
    - Grow 6x4 -> 8x5: widget at (3,2) -> (4,2) proportionally
    - Grow: spans stay same size (2x2 stays 2x2)
    - Shrink 8x5 -> 6x4: widget at (7,4) clamped to (4,2) (within bounds)
    - Overlap after remap: second widget nudged to adjacent free cell
    - No fit on page: widget spilled to next page at (0,0)
    - Min span exceeds grid: widget marked visible=false
    - Repeated resize (small->large->small): produces same layout as direct small (base snapshot prevents drift)
    - Edit mode: setGridDimensions during edit mode does NOT remap; remap applies on edit mode exit
    - Save updates base snapshot and saved dims
  </behavior>
