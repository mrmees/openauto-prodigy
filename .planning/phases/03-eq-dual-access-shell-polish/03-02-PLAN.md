**SettingsMenu.qml** — Add deep navigation function and guard the onVisibleChanged reset:

1. Add a property to track pending deep navigation:
   ```qml
   property bool _pendingEqNav: false
   ```

2. Add the `openEqDirect()` function:
   ```qml
   function openEqDirect() {
       _pendingEqNav = true
       if (visible) {
           _doEqNav()
       }
       // Otherwise onVisibleChanged will handle it
   }

   function _doEqNav() {
       _pendingEqNav = false
       if (settingsStack.depth > 1) {
           settingsStack.pop(null)
       }
       settingsStack.push(audioPage)
       ApplicationController.setTitle("Settings > Audio")
       Qt.callLater(function() {
           settingsStack.push(eqDirectComponent)
           ApplicationController.setTitle("Settings > Audio > Equalizer")
       })
   }
   ```

3. Add the EQ direct component (after the existing Component declarations around line 142):
   ```qml
   Component { id: eqDirectComponent; EqSettings {} }
   ```

4. **CRITICAL:** Guard the `onVisibleChanged` handler to not reset when deep navigation is pending:
   ```qml
   onVisibleChanged: {
       if (visible) {
           if (_pendingEqNav) {
               _doEqNav()
           } else {
               resetToGrid()
           }
       }
   }
   ```

5. **AudioSettings.qml** — Add `objectName: "Audio"` to the Flickable root element so the back handler can restore the correct title when popping from EQ (depth 3) to Audio (depth 2). The existing back handler reads `settingsStack.currentItem.objectName` for title restoration.
