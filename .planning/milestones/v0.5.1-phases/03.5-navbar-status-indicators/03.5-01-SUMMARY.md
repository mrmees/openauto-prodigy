## Decisions Made
- Battery parsed from ControlChannel unknownMessage 0x0017 rather than modifying ControlChannel.cpp -- keeps submodule changes minimal
- Signal strength requires submodule change (3 lines in PhoneStatusChannelHandler) -- noted for upstream push to open-android-auto
- hidden_ui_elements is the ONLY field set on AdditionalVideoConfig -- no ui_theme (known to break margins per Phase 03.4)
- Properties use -1 as sentinel for "no data" (not 0, which is a valid battery level and signal strength)
- Battery icon uses \uebd0 for battery_1_bar (verified in bundled MaterialSymbolsOutlined.ttf)
