       - Store `msg["theme"].toObject()` as pendingThemeJson_

       - Extract chunk index and base64 data
       - Validate: reject if index < 0 or data empty
       - Decode base64: `QByteArray::fromBase64(b64.toLatin1())`
       - Append to pendingWallpaperData_, increment receivedWallpaperChunks_
       - Log progress (debug level)
       - If receivedWallpaperChunks_ >= expectedWallpaperChunks_: call applyReceivedTheme()

    6. Implement applyReceivedTheme:
       - Extract theme name, seed from pendingThemeJson_
       - Parse light/dark JSON objects into QMap<QString, QColor> maps. The companion sends camelCase keys (primaryContainer, onSurface, etc.). Convert to hyphenated: iterate over JSON keys, insert a hyphen before each uppercase letter, lowercase the result. This handles the camelCase-to-hyphenated conversion generically. Example: "primaryContainer" -> "primary-container", "onSurface" -> "on-surface", "surfaceContainerHighest" -> "surface-container-highest".
       - Call `themeService_->importCompanionTheme(name, seed, dayColors, nightColors, pendingWallpaperData_)`
       - Send theme_ack: `{"type": "theme_ack", "accepted": true}` via client_ write + flush
       - Clear all pending state (wallpaper data, theme JSON, counters)

    7. In onClientDisconnected(), add cleanup: clear pendingWallpaperData_, pendingThemeJson_, reset counters.
