# Phase 03.2: Companion Theme Import - Context

**Gathered:** 2026-03-09
**Status:** Ready for planning

<domain>
## Phase Boundary

Receive full M3 color palettes and wallpaper images from the companion Android app over the authenticated TCP socket (port 9876). Expand ThemeService from 16 AA wire tokens to the full 34 M3 color roles. Companion themes appear as named entries in the theme picker, auto-switch on receive, and include per-theme wallpapers. AA wire theming (0x8011/0x8012) is disabled — companion app is the sole source of dynamic themes.

</domain>

<decisions>
## Implementation Decisions

### Palette Expansion: 16 → 34 M3 Roles
- Expand ThemeService to expose all 34 Material Design 3 color roles as Q_PROPERTYs
- New roles: primaryContainer, onPrimaryContainer, secondary, onSecondary, secondaryContainer, onSecondaryContainer, tertiary, onTertiary, tertiaryContainer, onTertiaryContainer, error, onError, errorContainer, onErrorContainer, surfaceDim, surfaceBright, surfaceContainerLowest, surfaceContainerHigh, surfaceContainerHighest, inversePrimary, onBackground, onSurfaceVariant, scrim (promoted from derived), shadow
- Note: some of these overlap with existing derived colors — reconcile during implementation
- All bundled themes (default, AMOLED, Ocean, Ember, connected-device) updated to specify all 34 roles
- AA wire tokens from the 16-token subset map directly into the 34-role palette where names match

### AA Wire Theming Disabled
- Remove AA wire token handling (0x8011 inbound handler, 0x8012 response) — Google's implementation doesn't work for our use case
- Connected-device theme stays in codebase (dormant) for potential future use
- `applyAATokens()` method remains but is no longer called from protocol path
- Phase 03.1 protocol code for 0x8011/0x8012 gets removed or disabled

### Theme Identity & Lifecycle
- Each companion theme appears as a named entry in the theme picker (e.g., "Midnight Blue")
- Same name from companion = overwrite existing theme (colors + wallpaper replaced)
- Different name = new theme entry
- Theme stored at `~/.openauto/themes/{slug}/theme.yaml` + `wallpaper.jpg`
- Users can delete companion-imported themes from settings
- Bundled themes cannot be deleted
- Deleting the active companion theme falls back to "Default Theme" automatically

### Auto-Switch Behavior
- Companion send always auto-switches the HU to the received theme — no manual activation needed
- This applies regardless of what theme was previously active (companion always wins)
- Silent apply — the visual change IS the feedback, no toast/notification
- Rationale: user explicitly triggered send from phone, so intent is clear

### Wallpaper Handling
- Wallpaper stored inside the theme directory: `~/.openauto/themes/{slug}/wallpaper.jpg`
- Matches bundled theme pattern (config/themes/default/wallpaper.jpg)
- Deleting a theme deletes its wallpaper — no orphaned files
- Companion theme wallpaper auto-applies (clears any user wallpaper override)
- Companion app pre-crops wallpaper to HU display resolution

### Display Info in hello_ack
- Add `display` field to hello_ack response: `{"width": W, "height": H}`
- Dimensions from `--geometry` flag when set (testing), otherwise QGuiApplication::primaryScreen()->geometry()
- Companion uses this to crop wallpaper to correct aspect ratio
- Field is optional — old companions without display awareness default to 1024x600

### Protocol Messages
- Companion sends `theme` message (34 M3 roles light + dark, wallpaper metadata) → N `theme_data` chunks (base64 JPEG) → HU sends `theme_ack`
- All messages use existing HMAC-SHA256 MAC verification (same as status messages)
- `theme_ack` sent after all chunks received and theme applied

### Claude's Discretion
- M3 role derivation formulas for bundled themes (computing all 34 from existing color palettes)
- Exact Q_PROPERTY naming (camelCase for QML) and internal key format (hyphens for YAML)
- How to handle partial M3 palettes (companion sends fewer than 34 roles)
- Theme slug generation from theme name
- Chunk reassembly buffer management
- Whether to keep `applyAATokens()` method signature or refactor for M3 roles
- Test coverage approach for new functionality

</decisions>

<specifics>
## Specific Ideas

- Design doc at `docs/plans/2026-03-09-companion-theme-import.md` has the full protocol spec and M3→AA mapping table
- The companion app sends camelCase M3 role names in JSON (primaryContainer, onSecondary, etc.) — ThemeService stores hyphens internally
- Companion wallpaper is always JPEG, pre-cropped by the app using `display` dimensions from hello_ack
- Phase 03 already handled all QML hardcode elimination — new M3 roles just add more Q_PROPERTYs for QML to optionally use

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `CompanionListenerService`: TCP server with challenge-response auth, MAC verification, message routing — add theme/theme_data handlers
- `ThemeService::applyAATokens()`: Takes day/night ARGB uint32 maps, merges into theme YAML, emits colorsChanged — adapt for 34 M3 roles
- `ThemeService::persistConnectedDeviceTheme()`: Writes colors to YAML — pattern reusable for companion theme persistence
- `ThemeService::scanThemeDirectories()`: Discovers themes in ~/.openauto/themes/ — companion themes auto-discovered
- `ThemeService::resolveWallpaper()`: Already checks {themedir}/wallpaper.jpg — companion wallpapers work with existing logic
- Existing HMAC verification in CompanionListenerService — reused for theme messages

### Established Patterns
- Theme YAML: id, name, version, font_family, day: {...}, night: {...} with hyphenated color keys
- Q_PROPERTYs in camelCase (textPrimary, onSurface) backed by hyphenated YAML keys (text-primary, on-surface)
- colorsChanged signal triggers all QML color rebinding automatically
- Theme switching via ThemeService::setTheme(id) with wallpaper resolution
- Connected-device theme hidden from picker via id check in scanThemeDirectories

### Integration Points
- `CompanionListenerService.hpp/cpp` — add theme state, handlers, ThemeService pointer
- `ThemeService.hpp/cpp` — expand to 34 M3 roles, add companion theme import method
- `main.cpp` — wire ThemeService to CompanionListenerService
- `config/themes/*.yaml` — update all bundled themes to 34-role palettes
- Theme settings QML — add delete action for user-imported themes

</code_context>
