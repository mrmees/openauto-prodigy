**Add properties at the top of the Flickable** (inside `root`):
```qml
property real _currentScale: {
    var v = ConfigService.value("ui.scale")
    if (v !== undefined && v !== null && Number(v) > 0) return Number(v)
    return 1.0
}
property real _previousScale: _currentScale
property bool _showRevert: false
property int _revertCountdown: 10
```

Also add a `Connections` block to keep `_currentScale` reactive:
```qml
Connections {
    target: ConfigService
    function onConfigChanged(path, value) {
        if (path === "ui.scale") {
            root._currentScale = (value !== undefined && value !== null && Number(value) > 0) ? Number(value) : 1.0
        }
    }
}
```

**Scale stepper row** — Builds a `[-] 1.0 [+] (reset)` layout. Insert after ReadOnlyField("Screen"), before SectionHeader("General"):

```qml
Item {
    Layout.fillWidth: true

    RowLayout {
        anchors.fill: parent
```

**Helper function** — Add to Flickable root:
```qml
function _applyScale(newVal) {
    _previousScale = _currentScale
    ConfigService.setValue("ui.scale", newVal)
    ConfigService.save()
    // Start safety revert countdown
    _showRevert = true
    _revertCountdown = 10
    scaleRevertTimer.restart()
}
```

In the same `DisplaySettings.qml`, add the safety revert mechanism. This is an overlay that appears after any scale change, counts down from 10 seconds, and reverts if the user doesn't confirm.

**Timer** — Add inside the Flickable root, after the Connections block:
```qml
Timer {
    id: scaleRevertTimer
    interval: 1000
    repeat: true
    onTriggered: {
        root._revertCountdown--
        if (root._revertCountdown <= 0) {
            // Time's up — revert
            scaleRevertTimer.stop()
            root._showRevert = false
            ConfigService.setValue("ui.scale", root._previousScale)
            ConfigService.save()
        }
    }
}
```

**Overlay** — Add as a sibling AFTER the ColumnLayout (still inside the Flickable is wrong — it must be outside the Flickable to overlay properly). Actually, the Flickable IS the root item. The overlay should be a child of the Flickable but positioned with `parent` anchors so it floats over the scrollable content. Use a `z: 10` to ensure it draws on top:

```qml
// Place this as last child of the Flickable root, after the ColumnLayout
Rectangle {
    id: revertOverlay
    visible: root._showRevert
    z: 10
    anchors.bottom: parent.bottom
    anchors.horizontalCenter: parent.horizontalCenter
    anchors.bottomMargin: UiMetrics.marginPage
    width: Math.min(parent.width - UiMetrics.marginPage * 2, 400)
    height: revertContent.implicitHeight + UiMetrics.gap * 2
    radius: UiMetrics.radius
    color: ThemeService.barBackgroundColor
    border.color: ThemeService.highlightColor
    border.width: 2

    Column {
        id: revertContent
        anchors.centerIn: parent
        spacing: UiMetrics.spacing
        width: parent.width - UiMetrics.gap * 2

        Text {
            width: parent.width
            text: "Keep this size?"
            font.pixelSize: UiMetrics.fontTitle
            font.weight: Font.DemiBold
            color: ThemeService.normalFontColor
            horizontalAlignment: Text.AlignHCenter
        }

        Text {
            width: parent.width
            text: "Reverting in " + root._revertCountdown + "s..."
            font.pixelSize: UiMetrics.fontBody
            color: ThemeService.descriptionFontColor
            horizontalAlignment: Text.AlignHCenter
        }

        Rectangle {
            width: Math.min(parent.width * 0.6, 200)
            height: UiMetrics.touchMin
            anchors.horizontalCenter: parent.horizontalCenter
            radius: UiMetrics.radius
            color: ThemeService.highlightColor

            Text {
                anchors.centerIn: parent
                text: "Keep this size"
                font.pixelSize: UiMetrics.fontBody
                font.weight: Font.DemiBold
                color: "white"
            }
            MouseArea {
                anchors.fill: parent
                onClicked: {
                    scaleRevertTimer.stop()
                    root._showRevert = false
                    // Scale stays as-is — already saved
                }
            }
        }
    }
}
```

**Behavior:** Every tap on [-], [+], or reset triggers `_applyScale()`, which saves the new scale, stores the previous value, and starts/restarts the 10-second countdown. If the user taps "Keep this size", the countdown stops and the current scale persists. If the countdown reaches 0, the scale reverts to `_previousScale`. Per Claude's discretion: trigger on ANY scale change (not just extreme values) — simpler logic, well-understood Windows pattern.

**No animation on scale change** per anti-pattern guidance in research — scale changes should be instant re-layout, not animated (would be visually chaotic with every UiMetrics token shifting).
