```qml
Connections {
    target: ConfigService

    }
}
```
This ensures `globalScale` → `scale` → all tokens update reactively when config changes.

**NavbarControl.qml** — Overhaul clock display for glanceability:

1. **Horizontal clock** (the `Text` element at line 56): Replace `font.pixelSize: UiMetrics.fontBody` with auto-sizing to fill control height:
```qml
font.pixelSize: Math.round(root.height * 0.75)
font.weight: Font.DemiBold
```
The 0.75 multiplier accounts for font ascenders/descenders, achieving visual ~80% fill of control area per user decision.

2. **Format string** — In the Timer's `onTriggered`, replace `"h:mm AP"` with:
```qml
var is24h = ConfigService.value("display.clock_24h") === true
var fmt = is24h ? "HH:mm" : "h:mm"
var timeStr = Qt.formatTime(new Date(), fmt)
```
`"HH:mm"` = 24h with leading zero (standard automotive). `"h:mm"` = 12h without AM/PM per user decision.

3. **Vertical clock characters** — Update the character splitting logic. Currently strips spaces (for "h:mm AP"). New logic for both formats:
```qml
// Build stacked chars: e.g., "12:34" → ["1","2",":","3","4"]
var chars = []
for (var i = 0; i < timeStr.length; i++) {
    chars.push(timeStr[i])
}
clockVertRepeater.model = chars
```
No space-stripping needed since neither format contains spaces now.

4. **Vertical clock styling** — In the Repeater delegate `Text`, change:
```qml
font.pixelSize: Math.round(root.width * 0.55)
font.weight: Font.DemiBold
```
Slightly smaller than horizontal (0.55 of width vs 0.75 of height) since characters stack. With 5 chars (e.g., "1","2",":","3","4") this keeps total height manageable. The colon character is narrower so will naturally compress.
