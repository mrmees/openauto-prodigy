# Phase 03.3 Plan 01: M3 Color Role Mapping Fix Summary

**Removed 7 deprecated custom Q_PROPERTYs from ThemeService and bulk-renamed ~170 QML references from legacy AA token names to M3 standard role names**

## Performance

- **Duration:** 7 min
- **Started:** 2026-03-09T18:54:42Z
- **Completed:** 2026-03-09T19:01:42Z
- **Tasks:** 2 (committed together as atomic change)
- **Files modified:** 40

## Accomplishments
- Removed 7 deprecated Q_PROPERTYs from ThemeService (textPrimary, textSecondary, red, onRed, yellow, onYellow, pressed)
- Bulk renamed ~170 QML references across 31 files to M3 standard names
- Cleaned deprecated keys from all 5 bundled theme YAMLs and static key sets
- Updated legacy v1 migration map to emit M3 role names instead of deprecated custom tokens
- Updated 42 unit tests including new deprecatedKeysReturnTransparent test

## Task Commits

Tasks 1 and 2 committed together (atomic -- C++ removal requires simultaneous QML rename):

1. **Task 1+2: Remove deprecated Q_PROPERTYs + Bulk QML rename** - `68ab11d` (feat)

## Files Created/Modified
- `src/core/services/ThemeService.hpp` - Removed 7 custom Q_PROPERTY declarations and accessor methods
- `src/core/services/ThemeService.cpp` - Removed pressed() method, cleaned knownAATokenKeys/knownM3Keys/allThemeKeys, updated legacy migration map
- `tests/test_theme_service.cpp` - Updated dayModeColors/nightModeColors to use onSurface(), removed deprecated assertions, replaced backwardCompat test with deprecatedKeysReturnTransparent
- `tests/data/themes/default/theme.yaml` - Removed deprecated keys
- `config/themes/{default,amoled,ocean,ember,connected-device}/theme.yaml` - Removed text-primary, text-secondary, red, on-red, yellow, on-yellow from day and night maps
- `qml/controls/NormalText.qml` - Default color changed from textPrimary to onSurface
- `qml/components/Navbar.qml` - pressed -> primaryContainer for power menu items
- `qml/components/NavbarControl.qml` - yellow -> tertiary for accent indicator
- 28 additional QML files with textPrimary->onSurface, textSecondary->onSurfaceVariant, red->error, onRed->onError renames

## Decisions Made
- Combined Task 1 and 2 into a single commit because removing Q_PROPERTYs without simultaneously renaming QML references would cause runtime binding failures
- Updated legacy v1 theme migration to map directly to M3 names (normal_font->on-surface, description_font->on-surface-variant, highlight->error/tertiary) instead of through deprecated custom tokens
- Kept barShadow as derived color (1 Q_PROPERTY, 50% black) -- still provides navbar border visual separation
