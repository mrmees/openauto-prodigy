**Fix 1 — Config persistence (MAJOR):**

In `src/core/YamlConfig.cpp`, function `initDefaults()`, add the missing default after line 21 (`root_["display"]["theme"] = "default";`):

```cpp
root_["display"]["wallpaper_override"] = "";
```

This is the root cause: `setValueByPath("display.wallpaper_override", ...)` validates against the defaults tree (lines 711-718 of YamlConfig.cpp). Without this entry, writes silently fail and wallpaper selection is lost.

**Fix 2 — Dynamic wallpaper refresh (MINOR):**

In `src/core/services/ThemeService.hpp`, add a public Q_INVOKABLE method near the existing `setWallpaperOverride`:

```cpp
Q_INVOKABLE void refreshWallpapers();
```

In `src/core/services/ThemeService.cpp`, implement it:

```cpp
void ThemeService::refreshWallpapers()
{
    buildWallpaperList();
}
```

`buildWallpaperList()` already emits the necessary property change signals (`availableWallpapersChanged` etc.), so QML ComboBoxes will update automatically.

Then in `qml/applications/settings/DisplaySettings.qml`, add a `Component.onCompleted` handler (or equivalent) to the wallpaper SettingsComboBox section that calls `ThemeService.refreshWallpapers()` when the settings page loads. This ensures new wallpapers are picked up each time the user opens Display settings, without needing a filesystem watcher.

Alternatively, if DisplaySettings.qml already has an `onCompleted` or `onVisibleChanged`, add the call there. The key point: call `ThemeService.refreshWallpapers()` when the wallpaper picker becomes visible.
