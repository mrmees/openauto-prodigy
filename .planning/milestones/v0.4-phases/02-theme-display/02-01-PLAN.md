  <name>Task 1: Extend ThemeService with multi-theme scanning and switching</name>
  <files>src/core/services/ThemeService.hpp, src/core/services/ThemeService.cpp, tests/test_theme_service.cpp</files>
  <behavior>
    - scanThemeDirectories(searchPaths) populates availableThemes (IDs) and availableThemeNames (display names)
    - scanThemeDirectories skips directories without theme.yaml
    - scanThemeDirectories deduplicates by theme ID (first wins)
    - setTheme(validId) loads the theme, emits colorsChanged(), returns true
    - setTheme(invalidId) returns false, does not change active theme
    - wallpaperSource returns "file:///path/to/wallpaper.jpg" if theme dir contains wallpaper.jpg, empty string otherwise
    - wallpaperSource updates when theme changes (wallpaperChanged signal)
    - availableThemesChanged signal emits after scan
  </behavior>
  <action>
Add three new Q_PROPERTYs to ThemeService.hpp:
- `availableThemes` (QStringList, READ availableThemes, NOTIFY availableThemesChanged) -- theme IDs
- `availableThemeNames` (QStringList, READ availableThemeNames, NOTIFY availableThemesChanged) -- display names
- `wallpaperSource` (QString, READ wallpaperSource, NOTIFY wallpaperChanged)

Add new signals: `availableThemesChanged()`, `wallpaperChanged()`.

Add private members:
- `QStringList availableThemes_`, `QStringList availableThemeNames_`
- `QMap<QString, QString> themeDirectories_` (maps theme ID to directory path)

Implement `scanThemeDirectories(const QStringList& searchPaths)`:
- For each search path, iterate subdirectories
- For each subdir containing `theme.yaml`, parse YAML to extract `id` and `name`
- Skip duplicates (first seen ID wins -- user themes searched before bundled, so user overrides)
- Populate `availableThemes_`, `availableThemeNames_`, `themeDirectories_`
- Emit `availableThemesChanged()`

Implement `setTheme(const QString& themeId)` (replace the stub):
- Look up `themeDirectories_[themeId]`
- If not found, return false
- Call existing `loadTheme(dirPath)` which already emits colorsChanged
- Resolve wallpaper: check for `wallpaper.jpg` in the theme dir. If exists, set wallpaperSource to `"file://" + absolutePath`. Else empty string.
- Emit `wallpaperChanged()` if wallpaper changed
- Return true

Implement `wallpaperSource()` getter: returns current wallpaper file:// URL or empty string.

Add tests to test_theme_service.cpp:
- `scanFindsMultipleThemes`: Create temp dirs with 3 themes, verify scan populates lists
- `scanSkipsMissingYaml`: Dir without theme.yaml is skipped
- `scanDeduplicatesById`: Same ID in two search paths, first wins
- `setThemeSwitchesColors`: setTheme to a different ID, verify backgroundColor changes
- `setThemeInvalidReturnsFalse`: setTheme with bogus ID returns false
- `wallpaperSourceResolvesFile`: Theme with wallpaper.jpg returns file:// URL
- `wallpaperSourceEmptyWhenNoImage`: Theme without wallpaper returns ""

For test themes: create temp directories with minimal theme.yaml files in each test (following the existing nightFallbackToDay pattern with QTemporaryDir).
