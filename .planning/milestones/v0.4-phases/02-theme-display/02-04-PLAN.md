Add a method to build the wallpaper list from scanned themes. Call this at the END of `scanThemeDirectories()`:
```cpp
void ThemeService::buildWallpaperList()
{
    availableWallpapers_.clear();
    availableWallpaperNames_.clear();

    // First entry: "None" (solid color background)
    availableWallpapers_.append("");
    availableWallpaperNames_.append("None");

    // Enumerate wallpapers from all scanned themes
    for (int i = 0; i < availableThemes_.size(); ++i) {
        const QString& themeId = availableThemes_[i];
        const QString& themeDir = themeDirectories_[themeId];
        QString wpPath = QDir(themeDir).filePath("wallpaper.jpg");
        if (QFile::exists(wpPath)) {
            QString fileUrl = "file://" + QFileInfo(wpPath).absoluteFilePath();
            availableWallpapers_.append(fileUrl);
            availableWallpaperNames_.append(availableThemeNames_[i]);
        }
    }

    emit availableWallpapersChanged();
}
```

Add `setWallpaper(const QString& wallpaperPath)` method:
```cpp
void ThemeService::setWallpaper(const QString& wallpaperPath)
{
    if (wallpaperSource_ == wallpaperPath) return;
    wallpaperSource_ = wallpaperPath;

}

Modify `setTheme()` so it does NOT automatically change the wallpaper. Currently, `setTheme()` resolves wallpaper.jpg from the theme dir. Change this: when `display.wallpaper` config key is empty or unset, `setTheme()` should still set the theme's default wallpaper. But when a wallpaper is explicitly selected by the user (stored in config), the user's choice should stick across theme changes.

The simplest approach: Remove the wallpaper resolution from `setTheme()` entirely. The wallpaper picker handles all wallpaper selection via `setWallpaper()`. The initial wallpaper is loaded from config in main.cpp (similar to how brightness is loaded). If no config value exists, use the current theme's wallpaper.jpg as default.

```

Wait — this creates a chicken-and-egg problem. Better approach: keep setTheme() setting the theme's default wallpaper, BUT also add an independent `setWallpaper()` that the picker calls. The config key `display.wallpaper` stores the user's explicit choice. On app startup (main.cpp), after `setTheme()`, check if `display.wallpaper` is set and override with that. This way:
- Switching theme sets the theme's default wallpaper (or empty if theme has none)
- User picking a wallpaper overrides with their choice and persists
- On restart, persisted wallpaper overrides theme default

Keep `setTheme()` wallpaper resolution as-is. Just add `setWallpaper()` for the picker. In main.cpp, after the setTheme call, add:

QString savedWallpaper = yamlConfig->valueByPath("display.wallpaper").toString();
