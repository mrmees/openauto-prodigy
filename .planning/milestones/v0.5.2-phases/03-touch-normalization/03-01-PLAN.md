**Create `qml/controls/SettingsRow.qml`:**

A container Item that provides alternating-row background for settings pages. API:

```
property int rowIndex: 0       // used to compute even/odd color
property bool interactive: false  // if true, show press feedback on tap
signal clicked()
default property alias content: contentContainer.data
```

- `Layout.fillWidth: true`, `implicitHeight: UiMetrics.rowH`
- Background Rectangle fills the item. Color alternates: even rows (`rowIndex % 2 === 0`) use `ThemeService.surfaceContainer`, odd rows use `ThemeService.surfaceContainerHigh`.
- If `interactive` is true, add a MouseArea over the whole row, and on press show a subtle state layer overlay (opacity 0.08 of `ThemeService.onSurface`). Wire MouseArea.clicked to root.clicked().
- Content goes inside a simple Item or RowLayout (`contentContainer`) that fills the row with `anchors.leftMargin: UiMetrics.marginRow` and `anchors.rightMargin: UiMetrics.marginRow`.
- No shadow, no border, no elevation -- flat row.

**Register in CMakeLists.txt:**

Add `set_source_files_properties(../qml/controls/SettingsRow.qml PROPERTIES QT_QML_SOURCE_TYPENAME "SettingsRow" QT_RESOURCE_ALIAS "SettingsRow.qml")` following the existing pattern (after ReadOnlyField registration, around line 141).

Add `../qml/controls/SettingsRow.qml` to the QML_FILES list (around line 327, after ReadOnlyField.qml).

**Update FullScreenPicker.qml for flat mode:**

Add `property bool flat: false`. When `flat` is true:
- Hide the shadow MultiEffect and rowBg Rectangle (the card background)
- Hide the state layer overlay Rectangle
- Remove the press scale animation (`scale` stays 1.0)
- The row content (RowLayout with label, current value, chevron icon) renders on transparent background
- The picker dialog itself stays unchanged (it's a modal overlay, not a row)

When `flat` is false (default), behavior is unchanged (backward compatible).

**Update SettingsToggle.qml, ReadOnlyField.qml, SettingsSlider.qml:**

These controls are already flat (no shadow/card). No visual changes needed. But verify that they:
- Use `UiMetrics.fontBody` for labels (already do)
- Do NOT use fontCaption or fontSmall for any visible label text (they don't)

**Update SettingsListItem.qml for flat mode:**

Add `property bool flat: false`. When `flat` is true:
- Hide the shadow MultiEffect and bg Rectangle
- Hide the state layer overlay
- Remove press scale animation
- The content (icon + label + chevron) renders on transparent background
- The MouseArea and clicked signal still work

**Update SegmentedButton.qml:**

No flat mode needed -- the segmented control sits on the right side of the row and its own styling (secondaryContainer segments) is part of its identity. Just verify fontSmall is acceptable for segment labels (it's a control element, not a settings label -- acceptable per decisions).

**Update SectionHeader.qml:**

Add `property int sectionIndex: -1` (unused by SectionHeader itself, but signals to consuming pages that the alternation counter should reset). No visual changes needed -- SectionHeader already has its own styling. The alternation reset is handled by the consuming page's index counter, not by SectionHeader.

Actually, simpler approach: SectionHeader doesn't need a property. Pages will just reset their row counter after each SectionHeader. Document this pattern.
