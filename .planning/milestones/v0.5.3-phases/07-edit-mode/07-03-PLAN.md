**1. Drop highlight rectangle (in gridContainer, outside Repeater):**
```qml
Rectangle {
    id: dropHighlight
    visible: false
    z: 5
    radius: UiMetrics.radius
    border.width: 2
    border.color: ThemeService.primary
    color: "transparent"
    opacity: 0.6
    // Position/size set dynamically during drag
    property bool isValid: true
    states: [
        State { name: "valid"; when: dropHighlight.isValid
            PropertyChanges { target: dropHighlight; border.color: ThemeService.primary } },
        State { name: "invalid"; when: !dropHighlight.isValid
            PropertyChanges { target: dropHighlight; border.color: ThemeService.error } }
    ]
}
```

**2. Per-delegate drag state properties:**
Inside the delegate Item, add:
```qml
property bool dragging: false
property real dragStartX: 0
property real dragStartY: 0
property real pressOffsetX: 0
property real pressOffsetY: 0
property real originalX: 0
property real originalY: 0
```

**3. Modify the per-widget MouseArea (currently z:-1, pressAndHoldInterval: 500):**
This MouseArea needs different behavior based on editMode:

When NOT in editMode:
- pressAndHold (500ms): enter editMode (existing Plan 02 behavior)

When IN editMode:
- pressAndHoldInterval: 200 (shorter hold before drag)
- onClicked: open context menu (Plan 02 behavior — tap)
- onPressAndHold: initiate drag
  - Set `dragging = true`
  - Record `originalX = delegate.x`, `originalY = delegate.y`
  - Record press offset: `pressOffsetX = mouse.x`, `pressOffsetY = mouse.y`
  - Change delegate visual: `opacity = 0.5`, `scale = 1.05`, raise z to 100
  - Show empty outlined placeholder at original position (a separate Rectangle in gridContainer, or use a property on the delegate to show outline-only mode at original pos)
  - `inactivityTimer.restart()`

- onPositionChanged:
  - If not dragging, return
  - Move delegate visually: compute gridContainer-space position from `mapToItem(gridContainer, mouse.x - pressOffsetX, mouse.y - pressOffsetY)` and set delegate x/y directly (this breaks the binding — that's intentional during drag)
  - Compute target cell: `var targetCol = Math.round(delegate.x / cellWidth)`, clamped to 0..gridColumns-colSpan
  - Same for targetRow
  - Check validity: `WidgetGridModel.canPlace(targetCol, targetRow, model.colSpan, model.rowSpan, model.instanceId)`
  - Update dropHighlight: position at `targetCol * cellWidth`, `targetRow * cellHeight`, size to `colSpan * cellWidth`, `rowSpan * cellHeight`, set isValid
  - Show dropHighlight
  - `inactivityTimer.restart()`

- onReleased:
  - If not dragging: return (was a tap, handled by onClicked)
  - Set `dragging = false`
  - Hide dropHighlight
  - Compute final target cell (same formula as onPositionChanged)
  - If `WidgetGridModel.moveWidget(model.instanceId, targetCol, targetRow)` succeeds:
    - Widget position updates automatically via model role bindings (delegate.x rebinds to model.column * cellWidth)
  - If moveWidget fails (returns false):
    - Animate delegate back to originalX/originalY, then rebind x/y to model roles
  - Reset delegate opacity to 1.0 and scale to 1.0, z back to normal
  - Hide placeholder

**4. Handling the binding break:**
When dragging starts, the delegate's x/y bindings (`x: model.column * cellWidth`) get broken by direct assignment. After drag ends (success or fail), re-establish the binding:
```qml
delegate.x = Qt.binding(function() { return model.column * gridContainer.cellWidth })
delegate.y = Qt.binding(function() { return model.row * gridContainer.cellHeight })
```
Do this after any snap-back animation completes.

Position/size it at the original grid cell when drag starts. Note: Qt Quick Rectangle doesn't support dashed borders natively — use a solid thin outline with reduced opacity.

- For all delegates: if any delegate.dragging is true, set dragging=false, rebind x/y, restore opacity/scale

- Simplest approach: track the currently-dragging instanceId at the homeScreen level. In exitEditMode, if a drag is active, clean it up.

Add to homeScreen:

Set it when drag starts, clear when drag ends. In exitEditMode, if non-empty, force cleanup.

- Do NOT use DragHandler — use MouseArea with manual position tracking

- Maintain finger offset (pressOffsetX/Y) so widget doesn't jump to finger center

**2. Add MouseArea to the resize handle (Plan 02 created the visual handle):**
Find the resize handle Rectangle in the delegate (bottom-right, visible in editMode). Give it a MouseArea child:

```qml
MouseArea {
    anchors.fill: parent
    // Expand touch area slightly for easier grabbing
    anchors.margins: -UiMetrics.spacing / 2

    property point startGlobal: Qt.point(0, 0)
    property int startColSpan: 0
    property int startRowSpan: 0
    property int proposedColSpan: 0
    property int proposedRowSpan: 0
    property bool resizing: false

    onPressed: function(mouse) {
        resizing = true
        startGlobal = mapToItem(gridContainer, mouse.x, mouse.y)
        startColSpan = model.colSpan
        startRowSpan = model.rowSpan
        proposedColSpan = startColSpan
        proposedRowSpan = startRowSpan

        // Position ghost at current widget bounds
        resizeGhost.x = model.column * gridContainer.cellWidth
        resizeGhost.y = model.row * gridContainer.cellHeight
        resizeGhost.width = model.colSpan * gridContainer.cellWidth
        resizeGhost.height = model.rowSpan * gridContainer.cellHeight
        resizeGhost.visible = true
        resizeGhost.isValid = true

        inactivityTimer.restart()
        mouse.accepted = true  // prevent event from reaching drag MouseArea
    }

    onPositionChanged: function(mouse) {
        if (!resizing) return
        var current = mapToItem(gridContainer, mouse.x, mouse.y)
        var deltaCols = Math.round((current.x - startGlobal.x) / gridContainer.cellWidth)
        var deltaRows = Math.round((current.y - startGlobal.y) / gridContainer.cellHeight)

        var newColSpan = startColSpan + deltaCols
        var newRowSpan = startRowSpan + deltaRows

        // Clamp to min/max constraints
        var wasAtLimit = false
        if (newColSpan < model.minCols) { newColSpan = model.minCols; wasAtLimit = true }
        if (newColSpan > model.maxCols) { newColSpan = model.maxCols; wasAtLimit = true }
        if (newRowSpan < model.minRows) { newRowSpan = model.minRows; wasAtLimit = true }
        if (newRowSpan > model.maxRows) { newRowSpan = model.maxRows; wasAtLimit = true }

        // Clamp to grid bounds
        if (model.column + newColSpan > WidgetGridModel.gridColumns)
            newColSpan = WidgetGridModel.gridColumns - model.column
        if (model.row + newRowSpan > WidgetGridModel.gridRows)
            newRowSpan = WidgetGridModel.gridRows - model.row

        proposedColSpan = newColSpan
        proposedRowSpan = newRowSpan

        // Check if proposed size overlaps other widgets
        var valid = WidgetGridModel.canPlace(model.column, model.row,
            newColSpan, newRowSpan, model.instanceId)
        resizeGhost.isValid = valid

        // Flash on boundary hit
        if (wasAtLimit) { /* trigger brief color animation */ }

        // Update ghost size
        resizeGhost.width = newColSpan * gridContainer.cellWidth
        resizeGhost.height = newRowSpan * gridContainer.cellHeight

        inactivityTimer.restart()
    }

    onReleased: {
        resizing = false
        resizeGhost.visible = false

        if (proposedColSpan !== startColSpan || proposedRowSpan !== startRowSpan) {
            // Atomic write on release (EDIT-09)
            WidgetGridModel.resizeWidget(model.instanceId, proposedColSpan, proposedRowSpan)
            // resizeWidget returns false if invalid — widget stays at original size
        }
    }
}
```

**3. Event ordering between resize handle and drag MouseArea:**
The resize handle MouseArea must consume the press event so the drag MouseArea (behind it at z:-1) doesn't also activate. Since the handle is at z:20 and the drag MouseArea is at z:-1, events should naturally hit the handle first. Use `mouse.accepted = true` in onPressed to be explicit. Also set `preventStealing: true` on the resize handle MouseArea to prevent the parent from stealing the drag gesture.

**4. Ghost color states:**
- Valid: `ThemeService.primary` border
- Invalid (overlap): `ThemeService.error` border
- At limit: Brief flash animation. Implement with a property `atLimit` that triggers a SequentialAnimation: change border color to a warning color for 200ms, then back to current state color. Use a Timer or SequentialAnimation on a helper property.

**5. Track resize state at homeScreen level:**
Add `property string resizingInstanceId: ""` to homeScreen. Set when resize starts, clear when it ends. In exitEditMode(), if non-empty, hide ghost and reset.

**Anti-patterns from research:**
- Do NOT animate the actual widget width/height during resize — ghost rectangle only (Pi performance)
- Do NOT call resizeWidget() during drag — only on release (EDIT-09 atomic writes)
