  <name>Task 1: Edit mode state, visual overlays, entry/exit, and FAB/remove interactions</name>
  <files>qml/applications/home/HomeMenu.qml</files>
  <action>
Rewrite HomeMenu.qml to add edit mode. This is the largest task in the phase but it's all one file and all QML. Key changes:

**1. Edit mode state property:**
Add `property bool editMode: false` on the homeScreen root Item.

**2. Inactivity timer:**
```qml
Timer {
    id: inactivityTimer
    interval: 10000
    onTriggered: homeScreen.editMode = false
}
```
Start the timer when editMode becomes true. Stop it when editMode becomes false.

**3. Edit mode reset function:**
Create a `function exitEditMode()` that sets `editMode = false`, hides any overlays (context menu, picker), and resets drag state (for Plan 03 compatibility). Call this from all exit paths.

**4. Long-press entry (replaces current behavior):**
Modify the background MouseArea (line 31-51 in current code): On `pressAndHold`, if NOT in editMode, set `editMode = true`. Do NOT open the picker directly anymore — that's done via the FAB now.

Modify the per-widget long-press MouseArea (line 101-113): Same change — when NOT in editMode, `pressAndHold` enters editMode instead of opening context menu. When IN editMode, short tap opens context menu (existing behavior), and hold+drag will be handled by Plan 03.

**5. Edit mode exit on tap outside:**
The background MouseArea also needs an `onClicked` handler: if `editMode` is true, call `exitEditMode()`. This naturally only fires on taps that don't hit a widget delegate (since delegates are on top).

**6. AA fullscreen auto-exit (EDIT-08):**
```qml
Connections {
    target: PluginModel
    function onActivePluginChanged() {
        if (PluginModel.activePluginFullscreen && homeScreen.editMode)
            homeScreen.exitEditMode()
    }
}
```

**7. Dotted grid lines overlay (visible in editMode):**
Inside gridContainer, add two Repeaters for vertical and horizontal dotted lines:
- Vertical: `model: WidgetGridModel.gridColumns + 1`, each delegate is a Column of small 1x2 Rectangles spaced at `UiMetrics.spacing` apart, positioned at `index * cellWidth`. Color: `ThemeService.outlineVariant`, opacity 0.3.
- Horizontal: same pattern, positioned at `index * cellHeight`.
- Both wrapped in an Item with `visible: homeScreen.editMode` and `z: 0` (below widgets).

**8. Widget delegate edit overlays:**
Inside the delegate's inner Item (the one with anchors.margins for gutter), add edit mode overlays:

a) **Accent border** — Rectangle with `anchors.fill: parent`, `color: "transparent"`, `border.width: 2`, `border.color: ThemeService.primary`, `radius: UiMetrics.radius`, `visible: homeScreen.editMode`, `z: 10`.

b) **X remove badge** — Small circle (diameter ~28) at top-right corner with red background (ThemeService.error) and white "X" MaterialIcon ("\ue5cd", size 16). Anchored to top-right of parent with small negative margins so it overlaps the corner. `visible: homeScreen.editMode`, `z: 20`. MouseArea inside that calls `WidgetGridModel.removeWidget(model.instanceId)` on click, with `inactivityTimer.restart()`.

c) **Resize handle** — Small triangle/square (UiMetrics.touchMin * 0.5) at bottom-right corner. Use a Rectangle with ThemeService.primary background, small radius. `visible: homeScreen.editMode`, `z: 20`. For now this is visual-only — Plan 03 adds the drag MouseArea on it.

**9. FAB (floating action button):**
After the gridContainer/LauncherDock ColumnLayout, add a "+" button:
- Rectangle, circular (radius = width/2), ~56x56 or `UiMetrics.touchMin * 1.2`
- Anchored to bottom-right of homeScreen with margins
- Above the LauncherDock (z: 50)
- Color: ThemeService.primary, with MaterialIcon "\ue145" (add icon) in ThemeService.onPrimary
- `visible: homeScreen.editMode`
- MouseArea onClicked:
  1. Restart inactivity timer
  2. Check if any widgets can fit: call `WidgetPickerModel.filterByAvailableSpace(WidgetGridModel.gridColumns, WidgetGridModel.gridRows)` then check if WidgetPickerModel.rowCount() > 0 (or check `findFirstAvailableCell(1,1)` for any space at all)
  3. If no space: `toast.show("No space available \u2014 remove a widget first")` (EDIT-10)
  4. If space: open pickerOverlay

**10. Picker modification for auto-place:**
Modify the picker's widget selection handler. Currently it places at `_targetCol/_targetRow`. Change it: after selecting a widget from the picker, use `WidgetGridModel.findFirstAvailableCell(defaultCols, defaultRows)` to find position. Use the widget's defaultCols/defaultRows (from WidgetPickerModel — may need to expose these). If findFirstAvailableCell returns -1, show toast. Otherwise call `placeWidget()` at the returned position. After placing, do NOT exit edit mode — user stays in edit mode to reposition if needed.

Fallback: if defaultCols/defaultRows aren't easily available from the picker model, use 2x2 clamped to grid dimensions as the default size (current behavior).

**11. Context menu behavior in edit mode:**
When editMode is true, a short TAP on a widget delegate should open the existing context menu. The per-widget MouseArea (currently long-press only) needs to also handle `onClicked` when in editMode. Add: `onClicked: { if (homeScreen.editMode) { requestContextMenu(); inactivityTimer.restart() } }`. The long-press handler in editMode will be used for drag initiation in Plan 03.

**12. Toast instance:**
Add a Toast component instance at the bottom of homeScreen (z: 200, above everything): `Toast { id: toast; anchors.bottom: parent.bottom; ... }`

**Important notes:**
- Restart `inactivityTimer` on EVERY user interaction (X badge tap, FAB tap, context menu open, picker open/close)
- When `editMode` transitions to false, stop the timer
- The per-widget long-press MouseArea at z:-1 should be careful: in non-edit mode it enters edit mode; in edit mode Plan 03 will add drag behavior. For now, in edit mode, it should do nothing on pressAndHold (Plan 03 will handle it).
