**HomeMenu.qml -- complete rewrite:**

Replace the entire 3-pane layout (RowLayout/ColumnLayout with WidgetHost instances, landscape/portrait branching, WidgetContextMenu/WidgetPicker Loaders, WidgetPlacementModel Connections) with:

1. A ColumnLayout filling the parent with `anchors.margins: UiMetrics.marginPage` and `spacing: UiMetrics.spacing`
2. Top slot: an Item (`id: gridContainer`) with `Layout.fillWidth: true` and `Layout.fillHeight: true`
3. Inside gridContainer: compute `cellWidth` and `cellHeight` as `readonly property real cellWidth: width / WidgetGridModel.gridColumns` (same for height/rows)
4. A Repeater with `model: WidgetGridModel` and an inline delegate Item:
   - `x: model.column * gridContainer.cellWidth`
   - `y: model.row * gridContainer.cellHeight`
   - `width: model.colSpan * gridContainer.cellWidth`
   - `height: model.rowSpan * gridContainer.cellHeight`
   - `visible: model.visible`
   - Do NOT use anchors on the delegate (conflicts with explicit x/y/width/height)
   - Inside: an inner Item with `anchors.fill: parent; anchors.margins: UiMetrics.spacing / 2` (half-spacing = 4px per side = 8px gutters between cards)
   - Inside inner: a glass card Rectangle (`anchors.fill: parent; radius: UiMetrics.radius; color: ThemeService.surfaceContainer; opacity: model.opacity`)
   - NO border Rectangle (user decision: drop the 1px outlineVariant border)
   - A Loader (`anchors.fill: parent; source: model.qmlComponent; asynchronous: false`)
5. Bottom slot: `LauncherDock { Layout.fillWidth: true }`
6. Keep `Component.onCompleted: ApplicationController.setTitle("Home")`
7. Remove ALL references to: WidgetPlacementModel, WidgetContextMenu, WidgetPicker, paneId, isLandscape branching, pane-based Connections

**WidgetHost.qml -- adapt for grid use:**

The grid currently inlines the glass card + Loader in the delegate, so WidgetHost is NOT used by the grid Repeater. However, WidgetHost.qml is still part of the QML module and may be referenced elsewhere. Adapt it to avoid WidgetPlacementModel errors:

1. Remove the `paneId` property
2. Remove the `isMainPane` property
3. Remove the `paneOpacity` property that calls `WidgetPlacementModel.paneOpacity(paneId)` -- replace with a simple `property real hostOpacity: 0.25`
4. Update the glass card Rectangle to use `hostOpacity` instead of `paneOpacity`
5. Remove the border Rectangle (no border per user decision)
6. Remove the empty state hint NormalText ("Hold to add widget")
7. Remove the Connections block targeting WidgetPlacementModel
8. Keep the Loader and longPressed signal/MouseArea (useful for future edit mode Phase 07)
