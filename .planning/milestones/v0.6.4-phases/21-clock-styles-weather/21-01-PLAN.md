**Part B: Rewrite ClockWidget.qml with style switching via Loader**

Restructure ClockWidget.qml to use a Loader that switches between three inline Component definitions based on `effectiveConfig.style`. Keep the existing digital clock as the `digitalComponent`. Add `analogComponent` and `minimalComponent`.

Key properties to add at the top of the root Item:
```qml
readonly property string clockStyle: currentEffectiveConfig.style || "digital"
```

Replace the current ColumnLayout with a Loader:
```qml
Loader {
    anchors.fill: parent
    sourceComponent: {
        switch (clockWidget.clockStyle) {
            case "analog": return analogComponent
            case "minimal": return minimalComponent
            default: return digitalComponent
        }
    }
}
```

**Digital component (id: digitalComponent):** Move the existing ColumnLayout with timeText, dateText, dayText into a Component block. Keep all existing behavior. The timer stays outside the Loader at the root level and updates properties that each component reads. Define shared properties at the root Item level:
- `property string currentTime: ""`
- `property string currentDate: ""`
- `property string currentDay: ""`

Update the existing Timer to set these root properties instead of setting text directly. Gate the timer on `isCurrentPage`:
```qml
Timer {
    interval: 1000
    running: clockWidget.isCurrentPage
    repeat: true
    triggeredOnStart: true
    onTriggered: {
        var now = new Date()
        var timeFmt = (clockWidget.timeFormat === "12h") ? "h:mm AP" : "HH:mm"
        clockWidget.currentTime = now.toLocaleTimeString(Qt.locale(), timeFmt)
        clockWidget.currentDate = now.toLocaleDateString(Qt.locale(), "MMMM d")
        clockWidget.currentDay = now.toLocaleDateString(Qt.locale(), "dddd")
    }
}
```

Digital component reads `clockWidget.currentTime`, `clockWidget.currentDate`, `clockWidget.currentDay`, `clockWidget.showDate`, `clockWidget.showDay`.

Since WidgetDescriptor.minCols/minRows are static (cannot vary by per-instance style), the QML must include a **small-size guard**: when the widget's colSpan < 2 OR rowSpan < 2, do NOT render the full clock face. Instead, show a centered MaterialIcon resize hint (e.g. `"\ue5c9"` open-in-full or `"\ue145"` add/expand) in `ThemeService.onSurfaceVariant` at `UiMetrics.fontHeading * 1.5`, signaling the user should resize. This prevents a squashed unreadable face at 1x1. Access span via `widgetContext.colSpan` and `widgetContext.rowSpan`.

At 2x2+ (colSpan >= 2 AND rowSpan >= 2), render the full analog face. Use ThemeService colors uniformly:

- Hour tick marks (12 ticks): `ThemeService.onSurfaceVariant`, lineWidth 2, from 85% to 95% of radius
- Hour hand: `ThemeService.onSurface`, lineWidth 3, 50% of radius
- Minute hand: `ThemeService.onSurface`, lineWidth 2, 70% of radius
- Second hand: `ThemeService.onSurfaceVariant`, lineWidth 1, 80% of radius
- Center dot: `ThemeService.onSurface`, radius 3px
- Date text below face at 2x2+: `ThemeService.onSurfaceVariant`, NormalText, `clockWidget.currentDate`

Structure the analog component as:
```qml
Component {
    id: analogComponent
    Item {
        anchors.fill: parent

        // Full analog face: only when 2x2+

            anchors.fill: parent

        }
    }
}
```
