**IMPORTANT DATA MODEL NOTE:** `widgetConfig()` returns only raw instance overrides. `effectiveWidgetConfig()` returns merged defaults + overrides. `setWidgetConfig()` expects only overrides (values that differ from defaults) -- NOT the full effective map. The config sheet MUST:
1. Read `effectiveWidgetConfig()` to show current values in controls (so defaults appear correctly)
2. Track which fields the user actually changes
3. Write ONLY the override map (changed values) via `setWidgetConfig()`, NOT the full effective map

This preserves the default/override distinction: if `defaultConfig` changes in a future version, only explicitly overridden fields keep their old values. Fields at default automatically pick up the new default.

    **CRITICAL: Override-only write path (Issue 3 fix):**

    When any control changes a value, the sheet must:
    1. Update `currentConfig[key]` for immediate display
    2. Mark the key in `overrideKeys[key] = true`
    3. Build an override-only map: iterate all keys in `overrideKeys`, include only those whose value in `currentConfig` differs from `defaultConfig[key]`. If a user changes a value back to the default, REMOVE it from overrides (it's no longer an override).
    4. Call `WidgetGridModel.setWidgetConfig(instanceId, overrideMap)` with ONLY the override map

    Helper function in the sheet:
    ```qml
    function applyConfig(key, value) {
        currentConfig[key] = value
        overrideKeys[key] = true

        // Build override-only map: only keys that differ from defaults
        var overrides = {}
        for (var k in overrideKeys) {
            if (overrideKeys[k] && currentConfig[k] !== defaultConfig[k]) {
                overrides[k] = currentConfig[k]
            }
        }
        WidgetGridModel.setWidgetConfig(root.instanceId, overrides)
    }
    ```

    This means: if `defaultConfig = {"format": "24h"}` and the user selects "24h" (the default), the override map will be empty `{}` -- the format key won't be persisted as an override. Only `{"format": "12h"}` would persist as an override. This preserves the default/override distinction for future-proofing.
