    **SettingsMenu.qml — Add custom StackView transitions:**

    Add all four transition properties to the StackView:
    ```qml
    StackView {
        id: settingsStack
        anchors.fill: parent
        initialItem: settingsList

        pushEnter: Transition {
            ParallelAnimation {
                OpacityAnimator { from: 0; to: 1; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
                XAnimator { from: settingsStack.width * 0.3; to: 0; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
            }
        }
        pushExit: Transition {
            ParallelAnimation {
                OpacityAnimator { from: 1; to: 0; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
                XAnimator { from: 0; to: -settingsStack.width * 0.3; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
            }
        }
        popEnter: Transition {
            ParallelAnimation {
                OpacityAnimator { from: 0; to: 1; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
                XAnimator { from: -settingsStack.width * 0.3; to: 0; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
            }
        }
        popExit: Transition {
            ParallelAnimation {
                OpacityAnimator { from: 1; to: 0; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
                XAnimator { from: 0; to: settingsStack.width * 0.3; duration: UiMetrics.animDuration; easing.type: Easing.OutCubic }
            }
        }
    }
    ```

    CRITICAL: Use ONLY Animator types (OpacityAnimator, XAnimator) inside the ParallelAnimation — mixing with NumberAnimation causes main-thread fallback.

    **Audit all settings sub-pages for root anchors:**

    Check every sub-page (DisplaySettings, AudioSettings, ConnectionSettings, VideoSettings, SystemSettings, CompanionSettings, AboutSettings, EqSettings). If any use `anchors.fill: parent` on their root element, StackView cannot animate `x` position.

    Current state: All sub-pages use `Flickable` as root. Flickable likely does NOT have `anchors.fill: parent` explicitly (it gets sized by StackView). But verify each file. If any sub-page root has `anchors.fill: parent` or `anchors.left/right/top/bottom`, remove root anchors and let StackView manage sizing (StackView automatically sizes items to fill).

    The `settingsList` Component (ListView) inside SettingsMenu uses `anchors.fill: parent` on the ListView. This is the initialItem — it should work since it's not animated in. But verify: if pop-back to the list looks wrong, wrap the ListView content differently.

    Also add `import QtQuick` at the top of SettingsMenu.qml if not already present (needed for OpacityAnimator/XAnimator types).
