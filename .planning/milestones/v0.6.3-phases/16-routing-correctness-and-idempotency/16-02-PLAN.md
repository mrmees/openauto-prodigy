Rewrite `_iptables_apply()` and `_iptables_flush()` and add `cleanup_stale_state()`. Specific changes:

**Constructor changes:**
- Add `redsocks_user` parameter (default `"redsocks"`) — the system user under which redsocks is spawned
- Change `default_skip_networks` default to `["127.0.0.0/8", "169.254.0.0/16", "10.0.0.0/24", "192.168.0.0/16"]` per user decision
- Change `default_skip_interfaces` default to `["lo", "eth0"]` per user decision (NOT wlan0 — redirecting AP outbound is the whole point)
- Remove `default_skip_ports` — port-based exemptions are replaced by proper owner/destination exemptions

**Redsocks spawning under dedicated user:**
- In `enable()`, spawn redsocks via `subprocess` with `user=` parameter (Python 3.9+ `asyncio.create_subprocess_exec` does not support `user=`, so use `su -s /bin/sh -c "redsocks -c {config_path}" {redsocks_user}` or `sudo -u {redsocks_user} redsocks -c {config_path}`). The cleanest approach: write the config file, then `await asyncio.create_subprocess_exec("sudo", "-u", self._redsocks_user, "redsocks", "-c", self._config_path)`. Since the daemon already runs as root, `sudo -u` works without password.

**New `_iptables_apply()` — deterministic flush/recreate (per user decision, NEVER check-and-reuse):**
1. Loop-delete all matching `OUTPUT -> OPENAUTO_PROXY` jumps until none remain:
   ```python
   while True:
       rc, _ = await self._run_cmd("iptables", "-t", "nat", "-D", "OUTPUT", "-p", "tcp", "-j", "OPENAUTO_PROXY")
       if rc != 0:
           break
   ```
2. Flush and delete existing chain (ignore errors — may not exist):
   ```python
   await self._run_cmd("iptables", "-t", "nat", "-F", "OPENAUTO_PROXY")
   await self._run_cmd("iptables", "-t", "nat", "-X", "OPENAUTO_PROXY")
   ```
3. Create fresh chain:
   ```python
   rc, out = await self._run_cmd("iptables", "-t", "nat", "-N", "OPENAUTO_PROXY")
   if rc != 0:
       raise RuntimeError(f"iptables -N failed: {out}")
   ```
4. Populate exemption rules in this EXACT order (per user decision):
   a. Owner exemption (redsocks user): `-A OPENAUTO_PROXY -m owner --uid-owner {redsocks_user} -j RETURN`
   b. Upstream SOCKS destination exemption (host+port, not host-only): `-A OPENAUTO_PROXY -d {proxy_host} -p tcp --dport {proxy_port} -j RETURN`
   c. Network exemptions: for each network, `-A OPENAUTO_PROXY -d {network} -j RETURN`
   d. Interface exemptions: for each interface, `-A OPENAUTO_PROXY -o {iface} -j RETURN`
   e. Final REDIRECT rule: `-A OPENAUTO_PROXY -p tcp -j REDIRECT --to-ports {redirect_port}`
5. Insert exactly one fresh OUTPUT jump: `-I OUTPUT -p tcp -j OPENAUTO_PROXY`

**New `_iptables_flush()` — loop-delete for completeness:**
1. Loop-delete all `OUTPUT -> OPENAUTO_PROXY` jumps (same while loop as above)
2. Flush chain: `iptables -t nat -F OPENAUTO_PROXY` (ignore error)
3. Delete chain: `iptables -t nat -X OPENAUTO_PROXY` (ignore error)

**`disable()` — cleanup failure honesty (per user decision: FAILED not DISABLED on error):**
- Wrap cleanup in try/except. Track which steps fail.
- If ANY cleanup step fails (iptables flush, DNS restore, redsocks stop): set state to `FAILED` with logging of which steps failed. Do NOT set `DISABLED`.
- Only set `DISABLED` when ALL cleanup steps succeed.
- Specifically:
  ```python
  errors = []
  try:
      await self._iptables_flush()
  except Exception as e:
      errors.append(f"iptables flush: {e}")
  try:
      await self._dns_restore()
  except Exception as e:
      errors.append(f"dns restore: {e}")
  # ... etc
  if errors:
      logging.error("Proxy cleanup failures: %s", "; ".join(errors))
      self._set_state(ProxyState.FAILED)
  else:
      self._set_state(ProxyState.DISABLED)
  ```

**New `cleanup_stale_state()` method:**
- Public async method. Called from `main()` before `ipc.start()`.
- Runs full proxy cleanup: kill orphaned redsocks, iptables flush, DNS restore. No detection — just clean.
- All errors are logged but do not raise — this is best-effort startup hygiene.
- Does NOT change state (state starts as DISABLED, stays DISABLED).
  ```python
  async def cleanup_stale_state(self) -> None:
      """Clean any stale proxy routing state from prior crash/restart."""
      logging.info("Cleaning stale proxy routing state...")
      # Kill any orphaned redsocks from prior abnormal exit.
      # The in-memory handle is lost across daemon restarts, so we must pkill.
      # If no redsocks is running, pkill returns non-zero — that's fine.
      try:
          await self._run_cmd("pkill", "-u", self._redsocks_user, "redsocks")
      except Exception as e:
          logging.warning("Stale redsocks cleanup error (non-fatal): %s", e)
      try:
          await self._iptables_flush()
      except Exception as e:
          logging.warning("Stale iptables cleanup error (non-fatal): %s", e)
      try:
          await self._dns_restore()
      except Exception as e:
          logging.warning("Stale DNS cleanup error (non-fatal): %s", e)
      # Clean stale config file
      try:
          os.remove(self._config_path)
      except FileNotFoundError:
          pass
      except Exception as e:
          logging.warning("Stale config cleanup error (non-fatal): %s", e)
      logging.info("Stale proxy state cleanup complete")
  ```
- **Why kill redsocks:** An abnormal restart (SIGKILL, power loss where ExecStopPost doesn't run) can leave an orphaned redsocks process. The in-memory `_redsocks_proc` handle is lost across daemon restarts. `enable()` starts a new redsocks, which will fail to bind the redirect port if the orphan still holds it. `pkill -u redsocks_user redsocks` is safe — if nothing is running, it's a no-op.

**Remove `skip_ports` parameter entirely** — port-based exemptions are superseded by proper owner/destination exemptions. Remove from constructor, `enable()` signature, and all internal usage.

**Keep `enable()` guard:** if state != DISABLED, call disable() first (existing behavior, still correct).
