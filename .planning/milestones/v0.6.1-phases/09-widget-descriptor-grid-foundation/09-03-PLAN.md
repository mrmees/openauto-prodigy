<objective>
Add YAML grid dimension persistence (grid_cols/grid_rows), rewrite setGridDimensions() with the proportional remap algorithm in WidgetGridModel, and establish the base snapshot architecture so that changing display/density preserves widget placements.

Purpose: GL-03 ensures users never lose widget placements when switching displays or adjusting grid density. The remap algorithm is the most algorithmically complex piece of Phase 09.
Output: YAML grid dim fields, remap algorithm with nudge/spill, base snapshot pattern, comprehensive unit tests.

**NOTE: This is the batch end for Plans 02+03.** Plan 02 changes DisplayInfo and QML grid frame but does NOT touch setGridDimensions() behavior. This plan rewrites setGridDimensions() with the full remap algorithm and owns full test suite verification for the entire batch.

**Boot sequence contract (critical for correctness):**
At startup, three pieces of data arrive at WidgetGridModel in sequence:
1. `setPlacements(placements)` — loads placement data from YAML. This ONLY stores placements, does NOT trigger remap.
2. `setSavedDimensions(cols, rows)` — sets the baseline grid dims from YAML. This ONLY stores savedCols_/savedRows_, does NOT trigger remap.
3. `setGridDimensions(cols, rows)` — called from QML once layout completes and available rect is known. This is the ONLY call that can trigger remap.

Remap MUST NOT run until ALL THREE have been provided. `setGridDimensions()` checks a readiness guard: if `basePlacements_` is empty AND `savedCols_ == 0`, the model is not yet initialized — just store the dims, emit the signal, and defer remap. The first `setGridDimensions()` call after both `setPlacements()` and `setSavedDimensions()` have been called will trigger the initial remap (or identity copy if dims match). This prevents:
- Remap running with empty placements (producing empty grid)
- Remap running with savedCols_=0 (division by zero in proportional math)
- Double remap on startup (once from stale data, once from real data)
</objective>

    Boot readiness guard in setGridDimensions():
    - `setGridDimensions()` is rewritten with the full remap algorithm (replacing the current destructive clamp-and-hide)
    - BEFORE attempting remap, check readiness: `if (basePlacements_.isEmpty() && savedCols_ == 0)` — model not yet initialized. In this case, just store cols_/rows_, emit signal, return (same as the Plan 02 stub behavior).
    - This guard ensures remap only runs after BOTH `setPlacements()` AND `setSavedDimensions()` have been called.

    - Once ready: if new dims match savedCols_/savedRows_, `livePlacements_ = basePlacements_` copy (identity). If dims differ, call `remapPlacements()`.

    Remap algorithm (private method `remapPlacements`):
    - Input: basePlacements_, savedCols_, savedRows_, newCols, newRows, registry_
    - For each base placement, in order (first-placed wins):
      1. Proportional position: newCol = qRound(baseCol * newCols / savedCols), newRow = qRound(baseRow * newRows / savedRows)
      2. Spans stay original size (do NOT scale)
      3. Clamp: col = min(col, max(0, newCols - colSpan)), row = min(row, max(0, newRows - rowSpan))
      4. Check min span vs grid: if descriptor.minCols > newCols or descriptor.minRows > newRows, mark visible=false, add warning, continue
      5. Check overlap with already-placed widgets on same page using occupancy grid
      6. If overlap, try spiral nudge: scan positions outward from target (right, down-right, down, down-left, left, up-left, up, up-right), increasing radius. First non-overlapping position within bounds wins.
      7. If no position found on page, spill to next page at position (0,0). Auto-create page if needed.
      8. Mark placed position in occupancy

    - Add `Q_INVOKABLE void setEditMode(bool editing)`
    - When `setGridDimensions()` is called during edit mode, queue the dims but don't remap
