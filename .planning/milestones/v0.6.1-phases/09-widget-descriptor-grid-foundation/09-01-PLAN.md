  <files>qml/components/WidgetPicker.qml</files>
  <action>
    Rewrite `WidgetPicker.qml` from the current flat horizontal card strip to a category-grouped layout per CONTEXT.md: "category headers above horizontal card rows."

    Structure — outer vertical scroll (ListView or Column+Flickable) of category sections, each containing:

    1. **"No Widget" entry at the top:** The first item in the model has empty categoryLabel. Render it as a standalone card/button before the category groups (not inside any category section). Style: icon + "No Widget" text, same card style as widget cards but full-width or centered, clearly distinct as a "clear slot" action.

    2. **Category sections:** For each category group (items sharing the same non-empty categoryLabel value):
       - **Category header:** `NormalText` with the category label (e.g. "Status", "Media"), using `UiMetrics.fontBody` size, `ThemeService.onSurfaceVariant` color, left-aligned with `UiMetrics.marginPage` left margin
       - **Horizontal card row:** A horizontal `ListView` (or `Row` inside `Flickable`) showing the widgets in that category as scrollable cards. Each card is a `Rectangle` with:
         - Rounded corners: `radius: UiMetrics.radius`
         - Background: `ThemeService.surfaceContainer`
         - Fixed card width (e.g., `UiMetrics.tileW * 0.6` or similar — executor discretion on exact sizing, just ensure icon+name+description fit)
         - Content: `MaterialIcon` with `icon: model.iconName` sized `UiMetrics.iconSize`, then `NormalText` for `model.displayName` (`UiMetrics.fontBody`, `ThemeService.onSurface`), then `NormalText` for `model.description` (`UiMetrics.fontSmall`, `ThemeService.onSurfaceVariant`)
         - Vertical stack layout within each card (icon on top or left, name, description below)
       - Cards spaced with `UiMetrics.spacing` between them

    Implementation approach — two options for the executor:
    - **Option A (Repeater-based):** Outer `Flickable` with `Column` containing a `Repeater` over a JS array of unique categoryLabels. Each delegate contains a header + inner horizontal `ListView` with a filtered proxy or manual JS filtering of the WidgetPickerModel by categoryLabel.
    - **Option B (Section-based):** Use `ListView` with `section.property: "categoryLabel"`, where each section delegate renders the header, and the main delegates are cards. However, since we want HORIZONTAL card rows per category (not a flat vertical list of cards), this requires wrapping: the section delegate would need to contain its own horizontal sub-list. This is more complex — Option A is likely simpler.

    The executor should choose whichever approach produces cleaner QML. The key requirement is: vertical list of categories, each with a header and a horizontal scrollable row of cards beneath it.

    Card click handler: call the existing picker selection callback (look at current WidgetPicker.qml for the onClicked pattern — likely emits a signal or calls a JS function).

    Use `UiMetrics` tokens for ALL sizes — no hardcoded pixels. Actual token names:
    - Corner radius: `UiMetrics.radius` (NOT cornerRadius)
    - Icon size: `UiMetrics.iconSize` (NOT iconMedium)
    - Body text: `UiMetrics.fontBody`
    - Description text: `UiMetrics.fontSmall` (NOT fontCaption)
    - Spacing: `UiMetrics.spacing`, `UiMetrics.gap`, `UiMetrics.marginPage`

    Preserve the existing picker overlay mechanics (how it's shown/hidden, how selection is communicated back to HomeMenu.qml). Only replace the visual layout, not the integration plumbing.
