Create `src/core/services/EqualizerService.hpp`:
- Inherits QObject and IEqualizerService
- Q_PROPERTY for mediaPreset, navigationPreset, phonePreset (QString, with NOTIFY signals)
- Q_INVOKABLE for all IEqualizerService methods
- `engineForStream(StreamId)` returns raw `EqualizerEngine*` for AudioStreamHandle assignment (Plan 02)
- Private: 3 `StreamState` structs each containing `EqualizerEngine engine` + `QString activePreset`
  - media: EqualizerEngine(48000.0f, 2)
  - navigation: EqualizerEngine(48000.0f, 1)
  - phone: EqualizerEngine(16000.0f, 1)
- Private: `QList<UserPreset> userPresets_` where UserPreset is `{QString name, std::array<float, kNumBands> gains}`
- Private: `QString generateAutoName() const` — finds lowest unused "Custom N"
- Note: No YamlConfig dependency in this plan. Config load/save added in Plan 02.

Create `src/core/services/EqualizerService.cpp`:
- Constructor: apply default presets (Flat for media, Voice for nav+phone) using `setAllGains()`
- `applyPreset`: look up name in bundled first, then user presets. If not found, fall back to Flat. Call `engine.setAllGains()` and update `activePreset`. Emit appropriate signal.
- `setGain`: call `engine.setGain(band, dB)`. Clear activePreset to "" (user has modified from preset). Emit gainsChanged.
- `saveUserPreset`: if name empty, use generateAutoName(). If name matches bundled preset, return "". Store current gains from the media stream (which stream? — the active/last-modified stream; simplest: save the gains for the most recently modified stream tracked via a `lastModifiedStream_` field, defaulting to Media). Actually per context discussion: save is always from the stream whose EQ the user is currently viewing. Since we don't have UI context here, take StreamId as parameter: `saveUserPreset(StreamId source, const QString& name = {})`. Emit presetListChanged.
- `deleteUserPreset`: remove from list. For each stream using this preset, revert to Flat. Emit signals.
- `renameUserPreset`: reject if new name matches bundled. Update name in list. Update any stream activePreset referencing old name. Emit presetListChanged.
- `generateAutoName`: iterate "Custom 1", "Custom 2", ... until no collision with existing user or bundled names.
