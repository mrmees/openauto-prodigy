Extend EqSettings.qml (created in Plan 01) to add preset management UI:

**1. Preset picker (bottom sheet dialog):**

Make the preset label in the control bar clickable. On click, open a `Dialog` styled as a bottom sheet (following the existing FullScreenPicker pattern: `parent: Overlay.overlay`, modal, `dim: true`, anchored to bottom).

Dialog content: `ListView` with a combined model assembled in JS:
```javascript
function buildPresetList() {
    var list = []
    var bundled = EqualizerService.bundledPresetNames()
    for (var i = 0; i < bundled.length; i++)
        list.push({ name: bundled[i], isUser: false, isSection: false })
    // Divider
    var userNames = EqualizerService.userPresetNames()
    if (userNames.length > 0) {
        list.push({ name: "User Presets", isUser: false, isSection: true })
        for (var j = 0; j < userNames.length; j++)
            list.push({ name: userNames[j], isUser: true, isSection: false })
    }
    return list
}
```

Use `ListModel` populated from the JS function. Rebuild on `EqualizerService.presetListChanged`.

Delegate:
- Section rows: Use `SectionHeader { text: model.name }` — not clickable
- Preset rows: Row with preset name text + checkmark icon (visible when `model.name === EqualizerService.activePreset(currentStream)`)
- User preset rows additionally support **swipe-to-delete**:
  - Content in a `Rectangle` with `x` property
  - Red delete background rectangle anchored to right, revealed by leftward swipe
  - `MouseArea` with `drag.target` (only for user presets), `drag.axis: Drag.XAxis`, `drag.minimumX: -deleteWidth`, `drag.maximumX: 0`
  - `onReleased`: if `contentRow.x < -deleteWidth / 2`, call `EqualizerService.deleteUserPreset(model.name)` and rebuild list; else snap back with animation
  - `Behavior on x { NumberAnimation { duration: 150 } }`
- Clicking a preset row: call `EqualizerService.applyPreset(currentStream, model.name)`, close dialog, refresh sliders

All text sizes via UiMetrics.fontBody / fontSmall. Row heights via UiMetrics.rowH. Touch targets must meet UiMetrics.touchMin.

**2. Preset label in control bar:**

Between the SegmentedButton and BYPASS badge, show the current preset name:
- Text: `EqualizerService.activePreset(currentStream) || "Custom"`
- Clickable — opens the preset picker dialog
- Append a small dropdown arrow icon (MaterialIcon "\ue5c5" or "\ue313")
- Update when `gainsChanged` or preset signals fire

**3. Save button:**

Add a save icon button (MaterialIcon "\ue161") in the control bar, near the preset label.
On click, open a save dialog:
- `Dialog` with `title: "Save Preset"`, modal
- `TextField` for preset name (placeholder: "Enter preset name")
- OK / Cancel buttons
- OK calls `EqualizerService.saveUserPreset(currentStream, textField.text)`, closes dialog
- Empty name: call `EqualizerService.saveUserPreset(currentStream, "")` for auto-name
- Virtual keyboard note: on Pi, QtVirtualKeyboard may or may not be available. The dialog should work either way — if no VK, user can still use auto-generated names (empty text = auto).

**4. Connect preset signals:**

Connect to `EqualizerService.presetListChanged` to rebuild preset list model.
Connect to the per-stream preset changed signals (`mediaPresetChanged`, `navigationPresetChanged`, `phonePresetChanged`) to update the preset label display.

All dimensions via UiMetrics. No hardcoded pixel values.

The `StackView.view` attached property on AudioSettings accesses the parent SettingsMenu's `settingsStack`. This allows EQ to be pushed as a depth-3 page (settingsList → audioPage → eqPage). The existing `onBackRequested` handler in SettingsMenu already pops when `depth > 1`, which handles this naturally.

If `StackView.view` doesn't work reliably (Qt 6.4 edge case), fall back to finding the StackView by walking up: use a `property var settingsStack` on AudioSettings and pass it from SettingsMenu when pushing. But try `StackView.view` first — it's the standard approach.

Show the media stream's current preset name as a subtitle/description on the right side of the row (before the chevron). This gives the user a quick glance at EQ state without opening the page.

Wrap the EqualizerService calls in `typeof EqualizerService !== "undefined"` checks to avoid errors when the service isn't available (matches existing patterns in AudioSettings for AudioService).

**SettingsMenu.qml:**

Update the `onBackRequested` handler to set the correct title when popping back from EQ page to Audio page. Currently it always resets to "Settings" on pop. With depth 3 (settings → audio → eq), popping from EQ should show "Settings > Audio", not "Settings".

Updated handler:
```javascript
function onBackRequested() {
    if (settingsStack.depth > 1) {
        settingsStack.pop()
        if (settingsStack.depth > 1) {
            // Popped back to a sub-page — reconstruct title
            // The generic approach: just use "Settings" with breadcrumb
            // For now, the audio sub-page case:
            ApplicationController.setTitle("Settings > Audio")
        } else {
            ApplicationController.setTitle("Settings")
        }
        ApplicationController.setBackHandled(true)
    }
}
```

Actually, a cleaner approach: store the title in the push and restore on pop. But that's more complex than needed. The simpler fix: when depth drops to 2 after pop, we know we're at a sub-page. The title should be "Settings > Audio" (the only sub-page that has sub-sub-pages). If more sub-sub-pages are added later, this can be generalized.

Alternatively, the simplest approach that works: keep the current handler as-is (always sets "Settings" on pop). The only issue is when popping from EQ → Audio, the title briefly shows "Settings" instead of "Settings > Audio". Fix: make the AudioSettings page set its own title on `Component.onCompleted` OR use `StackView.onCurrentItemChanged`.

**Simplest reliable approach:** Add `Component.onCompleted: ApplicationController.setTitle("Settings > Audio")` to AudioSettings.qml if not already present. When the StackView pops from EQ back to AudioSettings, the AudioSettings component becomes the current item again. If StackView reuses the component (doesn't destroy on push), the `onCompleted` won't re-fire. Instead, use the SettingsMenu's back handler with depth-aware title:

```javascript
function onBackRequested() {
    if (settingsStack.depth > 1) {
        settingsStack.pop()
        if (settingsStack.depth <= 1) {
            ApplicationController.setTitle("Settings")
        } else {
            // Sub-page is now visible again — set audio title
            ApplicationController.setTitle("Settings > Audio")
        }
        ApplicationController.setBackHandled(true)
    }
}
```

This works because the only depth-3 page (EQ) lives under Audio. If other sub-sub-pages are added later under different parents, this will need generalization (e.g., storing titles in a stack).
