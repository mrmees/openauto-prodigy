1. **`Q_INVOKABLE QVariantList gainsAsList(int streamIndex) const`** — converts `std::array<float, 10>` to QVariantList for QML. Takes `int` (not StreamId) since StreamId is not QML-registered. Cast internally: `auto sid = static_cast<StreamId>(streamIndex)`. Validate range 0-2.

2. **`bypassedChanged(int stream)` signal** — emit from `setBypassed()` so QML can react to bypass state changes. Add to the signals section.

3. **Update `setBypassed()`** to emit the new `bypassedChanged(int)` signal after calling `engine.setBypassed()`.

4. **`Q_INVOKABLE int bandCount() const`** — returns `kNumBands` (10). Convenient for QML Repeater model count without hardcoding.

5. **`Q_INVOKABLE QString bandLabel(int band) const`** — returns human-readable frequency label for a band index (0→"31", 1→"63", 2→"125", 3→"250", 4→"500", 5→"1k", 6→"2k", 7→"4k", 8→"8k", 9→"16k"). Centralizes the label mapping in C++ rather than duplicating in QML.

Do NOT modify the existing Q_INVOKABLE methods that take StreamId — they still work for C++ callers. The int-parameter overloads are additive. The existing `gainsForStream(StreamId)` stays for C++ use; `gainsAsList(int)` is the QML-friendly version.

IMPORTANT: The existing `gainsChanged(StreamId stream)` signal may not pass the StreamId value cleanly to QML (unregistered enum). Verify: if QML can receive it as an integer, great. If not, add a parallel `gainsChangedForStream(int stream)` signal and emit it alongside `gainsChanged()` in `applyPreset()` and `setGain()`. The safest approach is to emit both.

**EqBandSlider.qml** — Custom vertical slider for a single EQ band:

Properties:
- `property int bandIndex: 0`
- `property real gainValue: 0.0` (range -12.0 to +12.0)
- `property string freqLabel: ""`
- `property bool bypassed: false`

Signals:
- `signal gainChanged(real dB)`
- `signal resetRequested()`

Structure:
- Root `Item` with full column width as touch area
- Track: Thin vertical `Rectangle` centered horizontally (width ~4px via UiMetrics.spacing/2), full available height
- Thumb: `Rectangle` with rounded corners (height = UiMetrics.touchMin * 0.6, width = UiMetrics.touchMin * 0.8), positioned vertically based on gainValue. Thumb color: ThemeService.highlightColor
- 0 dB reference line: Horizontal `Rectangle` across the full width at the vertical center of the track, subtle (ThemeService.descriptionFontColor, opacity 0.3, height 1px)
- Frequency label: `Text` below the track area, using UiMetrics.fontTiny, ThemeService.descriptionFontColor
- Floating dB label: `Text` above the thumb, visible only while dragging (opacity animation, 150ms). Format: "+3.0" / "-1.5" / "0.0". Use UiMetrics.fontSmall, ThemeService.normalFontColor on a semi-transparent background rectangle (ThemeService.controlBackgroundColor, radius)
- Bypass dimming: `opacity: bypassed ? 0.4 : 1.0` on the track+thumb group, with `Behavior on opacity { NumberAnimation { duration: 200 } }`

Touch handling:
- `MouseArea` fills the entire root Item (full column width)
- `onPressed`: set `isDragging = false`, show floating label, update gain from Y position
- `onPositionChanged`: set `isDragging = true`, map `mouseY` to dB value: `dB = 12.0 - (mouseY - trackTopY) / trackHeight * 24.0`, snap to 0.5 dB (`Math.round(dB / 0.5) * 0.5`), clamp to [-12, 12], update `gainValue`, emit `gainChanged(dB)`
- `onReleased`: hide floating label
- `onDoubleClicked`: only if `!isDragging` — set `gainValue = 0`, emit `resetRequested()`

ALL dimensions must use UiMetrics — no hardcoded pixel values.
