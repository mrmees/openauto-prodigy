
**Step 2: Add sd_notify calls to main.cpp**

At the top, conditionally include:
```cpp
#ifdef HAS_SYSTEMD
#include <systemd/sd-daemon.h>
#endif
```

**READY notification:** After the QML engine successfully loads and the event loop is about to start (after `engine.load()` succeeds and QML objects are created), call:
```cpp
#ifdef HAS_SYSTEMD
sd_notify(0, "READY=1");
qCInfo(lcCore) << "sd_notify: READY=1 sent";
#endif
```

This should go just before `app.exec()` or right after the QML engine load success check. The READY signal tells systemd "I'm fully initialized, start dependent services." Place it after all plugins are initialized and QML is loaded — that's the "ready" state.

**WATCHDOG heartbeat:** Create a QTimer that fires at WatchdogSec/2 (15 seconds). Use `sd_watchdog_enabled(0, &usec)` to dynamically read the interval if available, otherwise hardcode 15s as fallback:

```cpp
#ifdef HAS_SYSTEMD
QTimer* watchdogTimer = new QTimer(&app);
uint64_t watchdogUsec = 0;
if (sd_watchdog_enabled(0, &watchdogUsec) > 0 && watchdogUsec > 0) {
    // Fire at half the watchdog interval
    int intervalMs = static_cast<int>(watchdogUsec / 1000 / 2);
    watchdogTimer->setInterval(intervalMs);
} else {
    watchdogTimer->setInterval(15000); // 15s fallback
}
QObject::connect(watchdogTimer, &QTimer::timeout, []{
    sd_notify(0, "WATCHDOG=1");
});
watchdogTimer->start();
qCInfo(lcCore) << "Watchdog heartbeat started (interval:" << watchdogTimer->interval() << "ms)";
#endif
```

Place this after the READY notification and before `app.exec()`.

**STOPPING notification:** Add a shutdown handler that sends `STOPPING=1`:
```cpp
#ifdef HAS_SYSTEMD
QObject::connect(&app, &QGuiApplication::aboutToQuit, []{
    sd_notify(0, "STOPPING=1");
    qCInfo(lcCore) << "sd_notify: STOPPING=1 sent";
});
#endif
```

**Important notes:**
- All sd_notify calls are `#ifdef HAS_SYSTEMD` guarded — the app compiles and runs fine without libsystemd (dev VM, non-systemd environments)
- `sd_notify(0, ...)` — first arg 0 means don't unset the environment variable (so subsequent calls work)
- The watchdog timer runs on the Qt main thread event loop — if the event loop freezes (deadlock), the timer won't fire, systemd detects it, kills and restarts the app. This is exactly the hang detection behavior we want.
- On dev VM builds, these calls are no-ops (HAS_SYSTEMD not defined or sd_notify returns harmlessly when not running under systemd)
