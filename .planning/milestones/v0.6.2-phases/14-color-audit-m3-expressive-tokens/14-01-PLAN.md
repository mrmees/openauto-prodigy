QColor ThemeService::activeColor(const QString& key) const {
    const bool night = nightMode();
    const auto& colors = night ? nightColors_ : dayColors_;
    auto it = colors.find(key);
    if (it != colors.end()) return it.value();
    if (night) { /* fallback to day */ }
    return QColor(Qt::transparent);
}

    6. Modify `activeColor()`: after looking up the color (before the `return`), add guardrail:
       ```
       if (nightMode() && isAccentRole(key)) {
           qreal sat = color.hslSaturationF();
           constexpr qreal kMaxNightSaturation = 0.55;
           if (sat > kMaxNightSaturation) {
               color.setHslF(color.hslHueF(), kMaxNightSaturation, color.lightnessF(), color.alphaF());
           }
       }
       ```
       Apply this to the looked-up color variable (make it non-const). Apply BEFORE the return, after both the primary lookup and the day-fallback lookup.

    IMPORTANT: The `activeColor()` guardrail must NOT apply to the tint computations' internal `activeColor("primary")` call, or the tint blend would double-clamp. However, since `surfaceTintHigh()`/`surfaceTintHighest()` call `activeColor()` which already applies the guardrail, the primary component is already clamped in night mode -- this is CORRECT behavior (the tint should use the clamped primary, not the raw one).
