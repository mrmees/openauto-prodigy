## Summary

This phase is a straightforward file migration: pull 9 companion-created theme directories from the Pi (`~/.openauto/themes/`), commit them into `config/themes/` in the repo, remove the 4 old bundled themes, and rename `prodigy` to `default`. The theme YAML format is already compatible -- ThemeService's `loadColorMap` uses `QColor(value)` which handles both `#RRGGBB` (old bundled) and `#ffRRGGBB` (companion ARGB) formats. No code changes are needed.

The main technical consideration is wallpaper JPEG sizes. The 9 new wallpapers total 8.1MB vs the current 24KB (3 tiny placeholders). The largest files (squares 1.8MB, ferns 1.9MB, retro 2.0MB) have dimensions of 4320x4320, 3000x3000, and 2667x2667 -- far larger than the 1024x600 display target. Resizing to 1440x1440 (matching prodigy's wallpaper) would cut repo bloat significantly while retaining more than enough quality for the target display. Git LFS is already active on the repo, so large binaries won't bloat history.

**Primary recommendation:** SCP theme directories from Pi, resize oversized wallpapers, adjust YAML metadata (id/name for prodigy->default rename), commit as bundled themes. Pure file operations, no code changes required.

## Architecture Patterns

### Current Bundled Theme Structure
```
config/themes/
  {theme-id}/
    theme.yaml       # Theme definition (id, name, version, font_family, day/night colors)
    wallpaper.jpg     # Optional wallpaper image
```

### Theme YAML Format (v2.0.0)

Two variants exist and both are fully compatible with ThemeService:

**Bundled format** (hand-crafted, `#RRGGBB`):
```yaml
id: default
name: Default Theme
version: 2.0.0
font_family: "Lato"
day:
  primary: "#e94560"
  # ... 34 M3 roles with comments
night:
  primary: "#c73650"
  # ... 34 M3 roles
```

**Companion format** (generated, `#AARRGGBB`):
```yaml
id: prodigy
name: prodigy
version: 2.0.0
source: companion
seed: "#5050F8"
font_family: Lato
day:
  background: "#fffcf8ff"
  # ... 34 M3 roles, no comments, alphabetical order
night:
  background: "#ff12121c"
  # ...
```

Key differences:
- Color format: `#RRGGBB` vs `#AARRGGBB` (both parsed by `QColor()`)
- Extra fields: `source: companion`, `seed: "#hex"` (ignored by ThemeService, harmless)
- Name field: lowercase vs title case
- Key ordering: semantic groups vs alphabetical
- Comments: present vs absent

### ThemeService Loading Path
1. `scanThemeDirectories()` reads `searchPaths` (user dir first, bundled second)
2. First-seen ID wins deduplication (user themes override bundled)
3. `connected-device` is registered but hidden from theme picker
4. Theme `name` field is displayed directly in the picker UI
5. `loadThemeFile()` reads id, name, font_family, then day/night color maps
6. `QColor(value)` handles all valid Qt color string formats

### Wallpaper Resolution
- `wallpaper.jpg` loaded by QML Wallpaper component with `sourceSize` capped to display dimensions
- Target display: 1024x600
- Wallpapers are square (companion app sends square crops from phone)
- Oversized wallpapers waste repo space but are downscaled at runtime anyway

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Theme YAML editing | Script to reformat YAMLs | Direct text edits (sed or manual) | Only 9 files; `id:` and `name:` fields on prodigy->default are the only changes needed |
| Wallpaper resizing | Custom image pipeline | `convert` (ImageMagick) or keep as-is | Simple one-shot resize; JPEG quality 85 at 1440x1440 is plenty for 1024x600 |
| Color format conversion | ARGB->RGB converter | Nothing -- keep ARGB as-is | `QColor()` handles both; converting adds risk of bugs for zero benefit |

## Common Pitfalls

### Pitfall 1: Renaming prodigy to default breaks theme loading
**What goes wrong:** If the directory is named `default/` but the YAML still says `id: prodigy`, ThemeService won't find it as the default fallback.
**Why it happens:** The fallback theme is looked up by ID `"default"`, not directory name.
**How to avoid:** Ensure the YAML `id:` field is changed to `default` AND the directory is named `default/`.

### Pitfall 2: Companion YAML has lowercase theme names
**What goes wrong:** Theme picker shows "prodigy" instead of "Prodigy" -- looks unprofessional.
**Why it happens:** Companion app stores the name as entered, typically lowercase.
**How to avoid:** Title-case the `name:` field in each theme.yaml when promoting to bundled.

### Pitfall 3: Wallpaper JPEGs bloat git history
**What goes wrong:** 8.1MB of binaries in git history forever, even if later replaced.
**Why it happens:** Git stores full copies of binary files on every change.
**How to avoid:** Git LFS is already active. Optionally resize the 3 largest wallpapers (squares 4320px, ferns 3000px, retro 2667px) down to ~1440px before committing. This would cut total from ~8.1MB to ~3-4MB.

### Pitfall 4: Forgetting to remove old themes
**What goes wrong:** Old themes (default, ember, ocean, amoled) remain alongside new ones, cluttering the theme picker.
**How to avoid:** Delete the 4 old theme directories before committing new ones.

### Pitfall 5: connected-device theme accidentally deleted
**What goes wrong:** If all of `config/themes/` is wiped and rebuilt, the connected-device internal theme disappears.
**How to avoid:** Explicitly preserve `config/themes/connected-device/` -- it's not in the removal list.
