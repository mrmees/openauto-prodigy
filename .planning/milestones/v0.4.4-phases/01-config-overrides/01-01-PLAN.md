  <action>
Rewrite `UiMetrics.qml` to implement the 3-tier override precedence chain. The existing singleton structure stays the same. Key changes:

1. **Add config reads** (bound once at creation time — do NOT call ConfigService.value() in computed properties):
```qml
readonly property var _cfgScale: ConfigService.value("ui.scale")
readonly property var _cfgFontScale: ConfigService.value("ui.fontScale")
```

2. **Add effective scale factors** (0 = not set = 1.0):
```qml
readonly property real globalScale: {
    var v = _cfgScale;
    if (v !== undefined && v !== null && Number(v) > 0) return Number(v);
    return 1.0;
}
readonly property real fontScale: {
    var v = _cfgFontScale;
    if (v !== undefined && v !== null && Number(v) > 0) return Number(v);
    return 1.0;
}
```

3. **Keep existing autoScale logic unchanged** (Phase 1 does NOT unconstrain the clamp — that's Phase 2):
```qml
readonly property real autoScale: {
    var shortest = Math.min(Screen.width, Screen.height);
    return Math.max(0.9, Math.min(1.35, shortest / 600));
}
```

4. **Rename existing `scale` to use combined logic:**
```qml
readonly property real scale: autoScale * globalScale
```

5. **Add combined font scale:**
```qml
readonly property real _fontScaleTotal: scale * fontScale
```

6. **Add token override helper** (returns NaN for "not set" — 0 is treated as "not set" since no valid token is 0px):
```qml
function _tok(name) {
    var v = ConfigService.value("ui.tokens." + name);
    if (v !== undefined && v !== null && Number(v) > 0) return Number(v);
    return NaN;
}
```

7. **Update token properties with override pattern:**

Layout tokens (use `scale`):
- `rowH`, `touchMin`, `headerH`, `iconSize`, `radius`, `tileW`, `tileH` — check `_tok(name)` first, fall back to `Math.round(base * scale)`

Font tokens (use `_fontScaleTotal`):
- `fontTitle`, `fontBody`, `fontSmall`, `fontHeading` — check `_tok(name)` first, fall back to `Math.round(base * _fontScaleTotal)`

Non-overridable tokens (scale-only, no individual override):
- `marginPage`, `marginRow`, `spacing`, `gap`, `sectionGap` — use `scale` directly (unchanged pattern)
- `fontTiny` — use `_fontScaleTotal` (benefits from fontScale but no individual override)
- `sectionH`, `backBtnSize`, `iconSmall`, `gridGap`, `tileIconSize` — use `scale` directly

Animation durations: unchanged (not scaled).

**Override precedence per the locked decisions:**
- Individual token override (`ui.tokens.fontBody: 24`) = absolute value, no multiplication
- `fontScale` stacks with `globalScale`: `fontBody = round(20 * autoScale * globalScale * fontScale)`
- `globalScale` stacks with `autoScale`: `rowH = round(80 * autoScale * globalScale)`

**CRITICAL: Use `{...}` block syntax for properties with override logic** — e.g.:
```qml
readonly property int rowH: { var o = _tok("rowH"); return isNaN(o) ? Math.round(80 * scale) : o; }
```
This is valid QML binding syntax. Properties without overrides keep the simple `Math.round(base * scale)` form.
